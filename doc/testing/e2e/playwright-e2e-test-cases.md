# Phase 10 前端验收用例

## 1. 文档目的

本文档给出 Phase 10 前端验收时真正需要执行的用例组。

用例来源：

- `doc/UI/new/flow/README.md`
- `doc/UI/new/pages/*.md`
- `doc/UI/new/dialogs/*.md`
- `doc/UI/new/shared/shared-components.md`
- `plan/archive/scrape-fail-graceful/plan.md`

本文档不再保留旧的假样本、旧的用例总数假设和与当前配置不一致的数据目录描述。

## 2. 全局前置条件

### 2.1 数据前置

先完成后端基线：

1. `test-data/scripts/seed-e2e-data.ps1 -SkipWorkerShutdown -NoPause`
2. `test-data/scripts/verify-backend-apis.ps1 -NoPause`

### 2.2 样本前置

前端验收默认围绕以下样本执行：

| 样本 | UI 角色 | 预期 |
|------|--------|------|
| `SNOS-037` | 成功样本 | 有海报、标题、演员 |
| `SDDE-759` | 成功样本 | 有海报、标题、演员 |
| `SDDE-660-C` | 识别样本 | 从 `sdde-660-c` 正常识别并展示为成功样本 |
| `FC2-PPV-1788676` | 失败样本 | 仍可见、显示占位图、详情页只呈现 stub sidecar 结果 |

### 2.3 前端环境前置

- 浏览器以 `?workerPort=` 或 `?workerUrl=` 连上 Worker
- `WorkerStatusOverlay` 已消失
- 主壳层已完成首次渲染

## 3. Main Shell Navigation

### TC-NAV-01 主壳结构完整

- 步骤：进入首页后观察主壳
- 断言：
  - 左侧存在 `设置 / 库管理 / 喜欢 / 演员`
  - 影视库列表区可见
  - 右侧内容区可承载页面切换

### TC-NAV-02 一级导航切换

- 步骤：依次点击 `设置`、`库管理`、`喜欢`、`演员`
- 断言：
  - 右侧内容区切换到对应页面
  - 左侧选中态与当前页面一致
  - 切换后不出现空白页或异常加载

### TC-NAV-03 库入口切换

- 步骤：点击左侧 `E2E-Lib-A`、`E2E-Lib-B`
- 断言：
  - 右侧切换到对应单库页
  - 页面上下文显示当前库
  - 从一个库切到另一个库后，不残留上一个库的数据

## 4. Library Management

### TC-LIB-01 库列表加载

- 步骤：进入库管理页
- 断言：
  - 至少出现 `E2E-Lib-A`、`E2E-Lib-B`
  - 每行能看到名称、影片数、状态与操作区

### TC-LIB-02 新建媒体库弹层

- 步骤：点击“新建媒体库”
- 断言：
  - 打开新建/编辑共用弹层
  - 弹层固定显示 3 条扫描目录输入
  - 底部按钮结构为 `取消 / 保存媒体库`

### TC-LIB-03 编辑媒体库弹层

- 步骤：点击任意库的“编辑”
- 断言：
  - 复用与新建相同的弹层结构
  - 当前库名与扫描目录已预填

### TC-LIB-04 删除媒体库确认

- 步骤：点击任意库的“删除”
- 断言：
  - 打开删除确认弹层
  - 弹层显示库摘要与“不删除磁盘影片文件”说明
  - 取消后列表不变

### TC-LIB-05 扫描反馈

- 步骤：点击任意库的“扫描”
- 断言：
  - 行内状态有反馈
  - 扫描进行中时，同库重复扫描入口不会无限重复触发

### TC-LIB-06 打开单库

- 步骤：点击任意库的“打开”或库名
- 断言：
  - 进入对应单库页
  - 返回后库管理页仍正常显示

## 5. Library Workbench

### TC-WB-01 结果集加载

- 步骤：进入 `E2E-Lib-A` 或 `E2E-Lib-B`
- 断言：
  - 影片卡片网格可见
  - 搜索、刷新、排序、分页区域可见

### TC-WB-02 筛选与返回恢复

- 步骤：修改搜索词、排序或分页后进入详情，再返回
- 断言：
  - 返回单库页后恢复原查询状态

### TC-WB-03 单卡菜单

- 步骤：对单张影片卡打开右键菜单或“更多”菜单
- 断言：
  - 菜单项至少覆盖 `播放 / 查看详情 / 收藏或取消收藏 / 重新抓取元数据 / 打开所在文件夹 / 删除影片原片`
  - 菜单结构与顺序符合 `doc/UI/new/dialogs/video-context-menu.md`

### TC-WB-04 详情进入与返回

- 步骤：点击影片卡进入详情，再返回
- 断言：
  - 进入影片详情页
  - 返回后恢复当前单库页上下文

### TC-WB-05 批量动作承载

- 步骤：勾选多张影片卡后观察批量动作入口
- 断言：
  - 多选后能承接批量收藏 / 批量重抓 / 清空选择 / 批量删除的语义
  - 若实现形态与正式文档不同，记录为待修复问题，而不是修改验收标准

## 6. Favorites

### TC-FAV-01 收藏结果加载

- 步骤：进入 Favorites
- 断言：
  - 收藏结果集可见或空态正确
  - 搜索、刷新、排序、分页区域与单库页语义一致

### TC-FAV-02 详情返回链路

- 步骤：从 Favorites 进入影片详情，再返回
- 断言：
  - 返回 Favorites
  - 返回后排序与分页状态恢复

### TC-FAV-03 单卡菜单与批量动作

- 步骤：在 Favorites 中打开单卡菜单并进入多选态
- 断言：
  - 复用与单库页一致的单卡菜单语义
  - 批量动作语义仍然可用

## 7. Actors

### TC-ACT-01 演员列表加载

- 步骤：进入 Actors
- 断言：
  - 演员列表可见
  - 搜索、排序、分页可用

### TC-ACT-02 进入演员详情

- 步骤：点击任意演员卡
- 断言：
  - 进入演员详情页
  - 可看到演员名称、Actor ID 和关联影片列表

### TC-ACT-03 列表返回恢复

- 步骤：从演员详情返回 Actors
- 断言：
  - 返回后 Actors 列表的搜索、排序、分页状态恢复

## 8. Actor Detail / Video Detail

### TC-ADV-01 演员详情关联影片

- 步骤：在演员详情页查看关联影片结果集
- 断言：
  - 关联影片卡片可见
  - 关联影片可继续进入详情

### TC-ADV-02 二级返回链路

- 步骤：`Actors -> Actor Detail -> Video Detail -> 返回 -> 返回`
- 断言：
  - 第一次返回回到 Actor Detail
  - 第二次返回回到 Actors

### TC-VD-01 影片详情基础显示

- 步骤：从任一结果页进入影片详情
- 断言：
  - 左侧海报/占位、VID、sidecar 状态、播放入口存在
  - 右侧标题、元数据、演员、路径信息存在

### TC-VD-02 演员入口

- 步骤：点击详情页演员标签
- 断言：
  - 进入对应演员详情
  - 返回后可回到当前影片详情

## 9. Settings

### TC-SET-01 六分组加载

- 步骤：进入设置页
- 断言：
  - 左侧显示 6 个分组：`基本 / 图片 / 扫描与导入 / 网络 / 库 / MetaTube`

### TC-SET-02 分组切换

- 步骤：依次切换 6 个分组
- 断言：
  - 右侧表单区域随分组变化
  - 切换过程中不出现崩溃或空白

### TC-SET-03 保存与恢复默认

- 步骤：修改设置、保存，再执行恢复默认
- 断言：
  - 保存成功后有反馈
  - 恢复默认后值回到默认态

### TC-SET-04 MetaTube diagnostics

- 步骤：切到 MetaTube 分组并执行 diagnostics
- 断言：
  - 请求能发出
  - 页面能展示 diagnostics 结果

### TC-SET-05 `settings.changed` 回流

- 步骤：保存设置后观察页面状态
- 断言：
  - 持久化值被刷新
  - 不会因为事件回流直接破坏当前页面状态

## 10. 抓取失败优雅降级专项

### TC-SFG-01 失败样本卡片显示

- 步骤：定位 `FC2-PPV-1788676`
- 断言：
  - 卡片显示 `No Poster` 占位图
  - 失败样本仍在结果集中可见

### TC-SFG-02 失败样本详情页

- 步骤：进入 `FC2-PPV-1788676` 详情
- 断言：
  - 页面可正常打开
  - sidecar 状态表现与 stub-only 样本一致
  - 不因缺图或缺元数据进入异常态

### TC-SFG-03 失败样本单卡重抓

- 步骤：在列表页或 Favorites 中对失败样本执行“重新抓取元数据”
- 断言：
  - 调用的是单影片搜刮，而不是全库搜刮
  - 列表页产生任务反馈或结果回刷

### TC-SFG-04 重抓后页面刷新

- 步骤：在失败样本重抓前后对比列表页和详情页
- 断言：
  - 若重抓成功，海报、状态、演员信息跟随更新
  - 若重抓仍失败，失败样本仍保持可见和可操作

### TC-SFG-05 正常识别样本显示

- 步骤：定位由 `sdde-660-c` 识别出的影片
- 断言：
  - UI 中按 `SDDE-660-C` 展示
  - 该样本具备成功抓取影片应有的海报、详情和演员信息

## 11. 人工降级项

以下项目在浏览器模式下只记录人工结论：

- 真实播放器是否启动
- 系统文件夹是否被真正打开
