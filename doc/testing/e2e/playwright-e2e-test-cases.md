# Phase 10 前端验收用例

## 1. 文档目的

本文档给出 Phase 10 前端验收时真正需要执行的用例组。

用例来源：

- `doc/UI/new/flow/README.md`
- `doc/UI/new/pages/*.md`
- `doc/UI/new/dialogs/*.md`
- `doc/UI/new/shared/shared-components.md`
- `plan/archive/scrape-fail-graceful/plan.md`

本文档不再保留旧的假样本、旧的用例总数假设和与当前配置不一致的数据目录描述。

## 2. 全局前置条件

### 2.1 数据前置

先完成后端基线：

1. `scripts/seed-e2e-data.ps1 -SkipWorkerShutdown -NoPause`
2. `scripts/verify-backend-apis.ps1 -NoPause`

### 2.2 样本前置

前端验收默认围绕以下样本执行：

| 样本 | UI 角色 | 预期 |
|------|--------|------|
| `SNOS-037` | 成功样本 | 有海报、标题、演员 |
| `SDDE-759` | 成功样本 | 有海报、标题、演员 |
| `SDDE-660-C` | 识别样本 | 从 `sdde-660-c` 正常识别并展示为成功样本 |
| `FC2-PPV-1788676` | 失败样本 | 仍可见、显示占位图、详情页只呈现 stub sidecar 结果 |

### 2.3 前端环境前置

- 浏览器以 `?workerPort=` 或 `?workerUrl=` 连上 Worker
- `WorkerStatusOverlay` 已消失
- 主壳层已完成首次渲染

## 3. Main Shell Navigation

### TC-NAV-01 主壳结构完整

- 步骤：进入首页后观察主壳
- 断言：
  - 左侧存在 `设置 / 库管理 / 喜欢 / 演员`
  - 影视库列表区可见
  - 右侧内容区可承载页面切换

### TC-NAV-02 一级导航切换

- 步骤：依次点击 `设置`、`库管理`、`喜欢`、`演员`
- 断言：
  - 右侧内容区切换到对应页面
  - 左侧选中态与当前页面一致
  - 切换后不出现空白页或异常加载

### TC-NAV-03 库入口切换

- 步骤：点击左侧 `E2E-Lib-A`、`E2E-Lib-B`
- 断言：
  - 右侧切换到对应单库页
  - 页面上下文显示当前库
  - 从一个库切到另一个库后，不残留上一个库的数据

已执行结果（2026-03-20）：

- 通过。当前浏览器模式下可稳定在 `E2E-Lib-A` 与 `E2E-Lib-B` 间切换，右侧内容区与左侧激活态同步更新。

## 4. Library Management

### TC-LIB-01 库列表加载

- 步骤：进入库管理页
- 断言：
  - 至少出现 `E2E-Lib-A`、`E2E-Lib-B`
  - 每行能看到名称、影片数、状态与操作区

### TC-LIB-02 新建媒体库弹层

- 步骤：点击“新建媒体库”
- 断言：
  - 打开新建/编辑共用弹层
  - 弹层固定显示 3 条扫描目录输入
  - 底部按钮结构为 `取消 / 保存媒体库`

已执行结果（2026-03-20）：

- 通过。实际弹层展示 3 条固定目录输入，按钮文案已对齐为 `保存媒体库`。

### TC-LIB-03 编辑媒体库弹层

- 步骤：点击任意库的“编辑”
- 断言：
  - 复用与新建相同的弹层结构
  - 当前库名与扫描目录已预填

已执行结果（2026-03-20）：

- 通过。`E2E-Lib-A` 编辑弹层已预填库名和路径 1。

### TC-LIB-04 删除媒体库确认

- 步骤：点击任意库的“删除”
- 断言：
  - 打开删除确认弹层
  - 弹层显示库摘要与“不删除磁盘影片文件”说明
  - 取消后列表不变

已执行结果（2026-03-20）：

- 通过。本轮补齐了摘要区，现展示名称、路径、影片数；取消后列表保持不变。

### TC-LIB-05 扫描反馈

- 步骤：点击任意库的“扫描”
- 断言：
  - 行内状态有反馈
  - 扫描进行中时，同库重复扫描入口不会无限重复触发

已执行结果（2026-03-20）：

- 通过。点击 `扫描` 后立即出现 `扫描任务已启动` toast，任务计数回刷正常；本轮未观察到重复触发。

### TC-LIB-06 打开单库

- 步骤：点击任意库的“打开”或库名
- 断言：
  - 进入对应单库页
  - 返回后库管理页仍正常显示

## 5. Library Workbench

### TC-WB-01 结果集加载

- 步骤：进入 `E2E-Lib-A` 或 `E2E-Lib-B`
- 断言：
  - 影片卡片网格可见
  - 搜索、刷新、排序、分页区域可见

### TC-WB-02 筛选与返回恢复

- 步骤：修改搜索词、排序或分页后进入详情，再返回
- 断言：
  - 返回单库页后恢复原查询状态

已执行结果（2026-03-20）：

- 通过。输入 `SDDE` 后进入 `SDDE-660-C` 详情并返回，仍停留在同一搜索条件。

### TC-WB-03 单卡菜单

- 步骤：对单张影片卡打开右键菜单或“更多”菜单
- 断言：
  - 菜单项至少覆盖 `播放 / 查看详情 / 收藏或取消收藏 / 重新抓取元数据 / 打开所在文件夹 / 删除影片原片`
  - 菜单结构与顺序符合 `doc/UI/new/dialogs/video-context-menu.md`

已执行结果（2026-03-20）：

- 通过。当前单卡菜单包含 `查看详情 / 播放 / 打开文件夹 / 收藏 / 重新抓取元数据 / 复制 VID / 删除影片`。
- 差异记录：实现额外提供了 `复制 VID`，同时当前菜单顺序是“查看详情”排在“播放”前，和规格顺序仍有偏差，保留到后续收口。

### TC-WB-04 详情进入与返回

- 步骤：点击影片卡进入详情，再返回
- 断言：
  - 进入影片详情页
  - 返回后恢复当前单库页上下文

### TC-WB-05 批量动作承载

- 步骤：勾选多张影片卡后观察批量动作入口
- 断言：
  - 多选后能承接批量收藏 / 批量重抓 / 清空选择 / 批量删除的语义
  - 若实现形态与正式文档不同，记录为待修复问题，而不是修改验收标准

已执行结果（2026-03-20）：

- 通过。当前单库页多选态已具备 `批量收藏 / 批量取消收藏 / 批量重抓 / 批量删除 / 取消选择`。
- 差异记录：承载形态仍是顶部动作条，不是规格要求的右键批量菜单。

## 6. Favorites

- 步骤：进入 Favorites
- 断言：
  - 收藏结果集可见或空态正确
  - 搜索、刷新、排序、分页区域与单库页语义一致

### TC-FAV-02 详情返回链路

- 步骤：从 Favorites 进入影片详情，再返回
- 断言：
  - 返回 Favorites
  - 返回后排序与分页状态恢复

### TC-FAV-03 单卡菜单与批量动作

- 步骤：在 Favorites 中打开单卡菜单并进入多选态
- 断言：
  - 复用与单库页一致的单卡菜单语义
  - 批量动作语义仍然可用

已执行结果（2026-03-20）：

- 通过。先在 `E2E-Lib-B` 将 `SDDE-660-C` 加入收藏，再进入 Favorites，可稳定看到 1 条结果与完整查询栏。
- 通过。单卡右键菜单包含 `查看详情 / 播放 / 打开文件夹 / 取消收藏 / 重新抓取元数据 / 复制 VID / 删除影片`。
- 通过。多选态出现 `批量取消收藏 / 重新抓取元数据 / 批量删除 / 取消选择`，取消选择后恢复普通点击进入详情。
- 通过。从 Favorites 进入 `SDDE-660-C` 详情再返回后，当前结果页保持稳定，未出现路由错乱或白屏。

## 9. Settings

### TC-SET-01 六分组加载

- 步骤：进入设置页
- 断言：
  - 左侧显示 6 个分组：`基本 / 图片 / 扫描与导入 / 网络 / 库 / MetaTube`

### TC-SET-02 分组切换

- 步骤：依次切换 6 个分组
- 断言：
  - 右侧表单区域随分组变化
  - 切换过程中不出现崩溃或空白

### TC-SET-03 保存与恢复默认

- 步骤：修改设置、保存，再执行恢复默认
- 断言：
  - 保存成功后有反馈
  - 恢复默认后值回到默认态

### TC-SET-04 MetaTube diagnostics

- 步骤：切到 MetaTube 分组并执行 diagnostics
- 断言：
  - 请求能发出
  - 页面能展示 diagnostics 结果

### TC-SET-05 `settings.changed` 回流

- 步骤：保存设置后观察页面状态
- 断言：
  - 持久化值被刷新
  - 不会因为事件回流直接破坏当前页面状态

已执行结果（2026-03-20）：

- 通过。6 个分组 `基本 / 图片 / 扫描与导入 / 网络 / 库 / MetaTube` 均可切换，右侧表单区跟随变化。
- 通过。在 MetaTube 分组将请求超时从 `60` 改为 `61` 后保存，收到 `设置已保存` toast，并观察到 `settings.changed` SSE 回流后的重新取值。
- 通过。点击 `恢复默认` 后，MetaTube 服务地址与超时值恢复默认态，并收到 `已恢复默认设置` toast。
- 通过。MetaTube diagnostics 可发起 `POST /api/settings/meta-tube/diagnostics` 并在页面展示成功摘要。
- 修复后通过。诊断结果展示已从错误的 `连接成功 (undefinedms)` 修正为真实摘要文本。

## 10. 抓取失败优雅降级专项

- 步骤：定位 `FC2-PPV-1788676`
- 断言：

- 步骤：进入 `FC2-PPV-1788676` 详情
