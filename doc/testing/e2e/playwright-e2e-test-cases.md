# Playwright E2E 自动化测试用例清单

## 文档目的

本文件从 `doc/UI/new/flow/` 下 7 张正式流程图逐一拆解，输出前端 E2E 自动化测试需要覆盖的具体用例。
每个用例包含前置条件、操作步骤、预期断言和已知限制。

本文件是 Phase 7.4 "Playwright 自动化回归" 的输入依据。

执行方式参见：`doc/testing/e2e/playwright-e2e-test-plan.md`

## 前置条件（全局）

| 条件 | 说明 |
|------|------|
| Worker 已启动 | `npm run tauri dev` 或手动启动 Worker，确保 `/api/app/bootstrap` 可访问 |
| 浏览器模式 | Playwright 通过 `http://localhost:1420?workerPort={port}` 进入 |
| WorkerStatusOverlay 已消失 | 主界面完成首次渲染，导航栏可见 |

## Flow 1：Main Shell Navigation Flow

> 来源：`main-shell-navigation-flow.png`

### TC-NAV-01：App Boot 后主壳结构完整

- **前置**：首次进入页面，WorkerStatusOverlay 消失
- **步骤**：
  1. `browser_snapshot` 获取页面元素树
- **断言**：
  - 左侧导航区可见
  - 导航项包含：设置、库管理、喜欢、演员
  - 右侧内容区可见
  - 影视库列表区域可见

### TC-NAV-02：点击"设置"导航

- **前置**：主壳已渲染
- **步骤**：
  1. `browser_click` 点击"设置"导航项
  2. `browser_snapshot` 获取新页面状态
- **断言**：
  - 右侧内容区切换为设置页
  - 设置页分组列表可见（基本 / 图片 / 扫描 / 网络 / 库 / MetaTube）

### TC-NAV-03：点击"库管理"导航

- **前置**：主壳已渲染
- **步骤**：
  1. `browser_click` 点击"库管理"导航项
  2. `browser_snapshot` 获取新页面状态
- **断言**：
  - 右侧内容区切换为库管理页
  - 库列表区域可见（可能为空列表或已有库）

### TC-NAV-04：点击"喜欢"导航

- **前置**：主壳已渲染
- **步骤**：
  1. `browser_click` 点击"喜欢"导航项
  2. `browser_snapshot` 获取新页面状态
- **断言**：
  - 右侧内容区切换为喜欢页
  - 影片卡片网格区域或空态提示可见

### TC-NAV-05：点击"演员"导航

- **前置**：主壳已渲染
- **步骤**：
  1. `browser_click` 点击"演员"导航项
  2. `browser_snapshot` 获取新页面状态
- **断言**：
  - 右侧内容区切换为演员页
  - 演员卡片网格区域或空态提示可见

### TC-NAV-06：导航切换后上下文保持

- **前置**：主壳已渲染
- **步骤**：
  1. 点击"设置"，进入设置页
  2. 点击"库管理"，进入库管理页
  3. 再次点击"设置"
  4. `browser_snapshot` 获取页面状态
- **断言**：
  - 设置页正确渲染，未出现空白或加载异常
  - 左侧导航当前选中项高亮正确

### TC-NAV-07：影视库列表入口——按库切换内容区

- **前置**：已有至少 1 个库存在
- **步骤**：
  1. 在左侧影视库列表中点击某个库名
  2. `browser_snapshot` 获取页面状态
- **断言**：
  - 右侧内容区切换为该库的 Library Workbench 页面
  - 页面标题或上下文反映所选库名

### TC-NAV-08：切换不同库时内容区跟随更新

- **前置**：已有至少 2 个库存在
- **步骤**：
  1. 点击库 A，等待内容加载
  2. 点击库 B
  3. `browser_snapshot` 获取页面状态
- **断言**：
  - 右侧内容区已更新为库 B 的内容
  - 不残留库 A 的数据

## Flow 2：Library Management Flow

> 来源：`library-management-flow.png`

### TC-LIB-01：进入库管理页加载库列表

- **前置**：主壳已渲染
- **步骤**：
  1. 点击"库管理"导航
  2. `browser_snapshot` 获取页面状态
- **断言**：
  - 页面完成加载（无 loading 状态残留）
  - 库列表正常渲染（空列表或已有库，无异常报错）

### TC-LIB-02：新建库

- **前置**：在库管理页
- **步骤**：
  1. 点击"新建"按钮
  2. `browser_snapshot` 确认弹层出现
  3. 在弹层中填写库名（如 `TestLib-{timestamp}`）
  4. 填写扫描目录
  5. 点击保存
  6. `browser_wait_for` 等待弹层消失
  7. `browser_snapshot` 获取最终状态
- **断言**：
  - 弹层正确打开，包含库名和扫描目录输入项
  - 保存后弹层关闭
  - 库列表中出现新建的库名
  - 新库显示初始状态信息

### TC-LIB-03：编辑库

- **前置**：至少存在 1 个库
- **步骤**：
  1. 对已有库点击编辑操作
  2. `browser_snapshot` 确认弹层出现
  3. 修改库名或扫描目录
  4. 点击保存
  5. `browser_snapshot` 获取最终状态
- **断言**：
  - 编辑弹层正确回填当前库信息
  - 保存后列表中库名已更新

### TC-LIB-04：删除库——二次确认

- **前置**：至少存在 1 个库
- **步骤**：
  1. 对已有库点击删除操作
  2. `browser_snapshot` 确认删除确认弹层出现
  3. 点击确认删除
  4. `browser_wait_for` 等待弹层消失
  5. `browser_snapshot` 获取最终状态
- **断言**：
  - 删除前弹出二次确认弹层
  - 确认后库从列表中消失
  - 列表正确刷新

### TC-LIB-05：删除库——取消

- **前置**：至少存在 1 个库
- **步骤**：
  1. 对已有库点击删除操作
  2. `browser_snapshot` 确认删除确认弹层出现
  3. 点击取消
  4. `browser_snapshot` 获取最终状态
- **断言**：
  - 弹层关闭
  - 库依然存在于列表中

### TC-LIB-06：保存后启动扫描任务——任务进度反馈

- **前置**：新建或编辑库后触发扫描
- **步骤**：
  1. 新建一个库并保存
  2. 观察库行内状态与全局活动条
  3. `browser_snapshot` 获取状态
- **断言**：
  - 行内状态反映扫描进行中或已完成
  - 全局活动条区域有任务反馈（如可见）

### TC-LIB-07：打开单库——进入 Library Workbench

- **前置**：至少存在 1 个库
- **步骤**：
  1. 在库管理页点击某个库的"打开"操作
  2. `browser_snapshot` 获取页面状态
- **断言**：
  - 页面切换为 Library Workbench（单库内容页）
  - 页面上下文反映所选库

### TC-LIB-08：重复库名冲突

- **前置**：已存在一个库（如 `TestLib`）
- **步骤**：
  1. 点击新建
  2. 填写与已有库同名的库名
  3. 点击保存
  4. `browser_snapshot` 获取状态
- **断言**：
  - 保存失败，弹层显示冲突 / 错误提示
  - 库列表不出现重复库

## Flow 3：Library Workbench Flow

> 来源：`library-workbench-flow.png`

### TC-WB-01：进入单库页加载影片列表

- **前置**：从库管理页或左侧库列表进入某个库
- **步骤**：
  1. 进入单库页
  2. `browser_snapshot` 获取页面状态
- **断言**：
  - 影片卡片网格正常渲染（有影片时显示卡片，无影片时显示空态）
  - 筛选 / 排序控件可见

### TC-WB-02：筛选与排序状态保持

- **前置**：单库内有影片数据
- **步骤**：
  1. 修改排序条件（如按时间倒序）
  2. `browser_snapshot` 确认排序生效
  3. 导航到其他页面
  4. 返回该单库页
  5. `browser_snapshot` 获取状态
- **断言**：
  - 返回后排序条件恢复为之前的设置
  - 分页状态一致

### TC-WB-03：查看影片详情——进入与返回

- **前置**：单库内有至少 1 部影片
- **步骤**：
  1. 点击某张影片卡片（查看详情）
  2. `browser_snapshot` 确认进入影片详情页
  3. 点击返回
  4. `browser_snapshot` 获取状态
- **断言**：
  - 正确进入影片详情页，显示海报/元数据
  - 返回后恢复到单库页，筛选/排序/分页状态不丢失

### TC-WB-04：右键 / 更多菜单——播放

- **前置**：单库内有至少 1 部影片
- **步骤**：
  1. 对影片卡片右键或点击更多菜单
  2. `browser_snapshot` 确认菜单弹出
  3. 点击"播放"
- **断言**：
  - 右键/更多菜单可见，包含播放选项
  - 播放操作触发（API 调用发出）
- **已知限制**：实际播放依赖 Tauri shell API，Playwright 环境中无法验证外部播放器启动，仅验证 API 调用

### TC-WB-05：右键 / 更多菜单——收藏与取消收藏

- **前置**：单库内有至少 1 部影片
- **步骤**：
  1. 对未收藏影片打开菜单，点击"收藏"
  2. `browser_snapshot` 确认收藏状态更新
  3. 再次打开菜单，点击"取消收藏"
  4. `browser_snapshot` 确认状态回退
- **断言**：
  - 收藏后卡片或菜单反映已收藏状态
  - 取消收藏后状态恢复

### TC-WB-06：右键 / 更多菜单——重新抓取元数据

- **前置**：单库内有至少 1 部影片
- **步骤**：
  1. 对影片打开菜单，点击"重新抓取"
  2. `browser_snapshot` 观察任务状态
- **断言**：
  - 触发元数据重新抓取任务
  - 行内摘要或全局活动条反映任务进度
  - 任务完成后状态回刷

### TC-WB-07：右键 / 更多菜单——打开所在文件夹

- **前置**：单库内有至少 1 部影片
- **步骤**：
  1. 对影片打开菜单，点击"打开文件夹"
- **断言**：
  - 菜单中"打开文件夹"选项可见
- **已知限制**：实际打开依赖 Tauri shell API / 系统外部能力，Playwright 环境中无法验证文件管理器打开，仅验证菜单项存在

### TC-WB-08：右键 / 更多菜单——删除原片

- **前置**：单库内有至少 1 部影片（建议使用测试数据）
- **步骤**：
  1. 对影片打开菜单，点击"删除原片"
  2. `browser_snapshot` 确认删除确认弹层出现
  3. 点击确认
  4. `browser_wait_for` 等待操作完成
  5. `browser_snapshot` 获取最终状态
- **断言**：
  - 删除前弹出确认弹层
  - 确认后影片从列表消失
  - 列表计数更新
  - 演员关联信息联动刷新（如有关联演员）
- **数据风险**：此用例会实际删除文件，执行时需使用专门的测试数据

## Flow 5：Actors Flow

### TC-ACT-01：进入演员列表页加载数据

- **前置**：主壳已渲染
- **步骤**：
  1. 点击"演员"导航

  - 无演员时显示空态

### TC-ACT-02：演员列表搜索

- **前置**：有演员数据
- **步骤**：
  1. 在搜索框输入演员名关键词
  2. `browser_snapshot` 确认搜索结果
- **断言**：
  - 搜索结果只显示匹配的演员
  - 清空搜索后恢复全部列表

### TC-ACT-03：演员列表排序与分页

- **前置**：有多位演员
- **步骤**：
  1. 修改排序条件
  2. `browser_snapshot` 确认排序生效
  3. 若有多页，切换分页
- **断言**：
  - 排序结果符合预期
  - 分页切换正确

### TC-ACT-04：点击演员卡片进入演员详情

- **前置**：有至少 1 位演员
- **步骤**：
  1. 点击某位演员卡片
  2. `browser_snapshot` 确认进入演员详情页
- **断言**：
  - 进入演员详情页
  - 显示演员基本信息
  - 加载关联影片（作品集）

### TC-ACT-05：演员详情——点击关联影片进入影片详情

- **前置**：演员详情页有关联影片
- **步骤**：
  1. 在演员详情页点击某部关联影片
  2. `browser_snapshot` 确认进入影片详情
- **断言**：
  - 进入影片详情页
  - 保留二级返回链路

### TC-ACT-06：二级返回链路——Video Detail → Actor Detail → Actors

- **前置**：从演员详情进入了影片详情
- **步骤**：
  1. 在影片详情页点击返回
  2. `browser_snapshot` 确认返回到演员详情页
  3. 在演员详情页点击返回
  4. `browser_snapshot` 确认返回到演员列表页
- **断言**：
  - 影片详情 → 演员详情：正确返回
  - 演员详情 → 演员列表：正确返回
  - 演员列表的搜索/排序/分页状态恢复

## Flow 6：Video Detail Playback Flow

> 来源：`video-detail-playback-flow.png`

### TC-VD-01：从结果页进入影片详情

- **前置**：从单库页 / 喜欢页 / 演员详情有至少 1 部影片
- **步骤**：
  1. 点击某张影片卡片
  2. `browser_snapshot` 确认进入详情页
- **断言**：
  - 页面渲染海报区域
  - 显示影片元数据
  - 显示演员入口（如有）
  - 记录 `backTo` 来源页

### TC-VD-02：播放按钮调用

- **前置**：在影片详情页
- **步骤**：
  1. 点击播放按钮
  2. `browser_snapshot` 获取状态
- **断言**：
  - 播放 API 调用发出（`POST /api/videos/{id}/play`）
  - 播放写回成功后，最近播放状态更新
- **已知限制**：实际播放器启动依赖 Tauri shell API，仅验证 API 调用和状态写回

### TC-VD-03：点击演员标签进入 Actor Detail

- **前置**：影片详情页有演员标签
- **步骤**：
  1. 点击某个演员标签
  2. `browser_snapshot` 确认进入演员详情页
- **断言**：
  - 进入对应演员的详情页
  - 当前影片详情状态不丢失（可通过返回验证）

### TC-VD-04：外链能力——打开外部来源

- **前置**：影片详情页有外部来源链接
- **步骤**：
  1. 点击外部来源链接
- **断言**：
  - 外链操作触发
  - 当前详情页状态不丢失
- **已知限制**：打开外部浏览器依赖系统能力，Playwright 环境仅验证链接元素存在与可点击

### TC-VD-05：返回来源页——状态恢复

- **前置**：从某个来源页（单库页 / 喜欢页）进入影片详情
- **步骤**：
  1. 在影片详情页点击返回
  2. `browser_snapshot` 获取页面状态
- **断言**：
  - 返回到正确的来源页
  - 来源页的筛选/排序/分页状态恢复

- **前置**：在设置页
- **步骤**：
  1. 依次点击各个分组（基本 → 图片 → 扫描 → 网络 → 库 → MetaTube）
  2. 每次切换后 `browser_snapshot` 获取状态
- **断言**：
  - 每个分组切换后右侧表单区域更新
  - 分组选中状态高亮正确
  - 不出现加载错误

### TC-SET-03：修改设置——Dirty 状态

- **前置**：在设置页某个可编辑分组
- **步骤**：
  1. 修改某个表单字段值
  2. `browser_snapshot` 获取状态
- **断言**：
  - 表单进入 Dirty 状态
  - 保存按钮变为可用 / 高亮

### TC-SET-04：保存设置

- **前置**：已修改设置（Dirty 状态）
- **步骤**：
  1. 点击保存按钮
  2. `browser_wait_for` 等待保存反馈
  3. `browser_snapshot` 获取状态
- **断言**：
  - 保存成功反馈（toast / 状态指示）
  - Dirty 状态清除
  - 刷新页面后值依然持久化

### TC-SET-05：保存设置后值持久化验证

- **前置**：已保存设置
- **步骤**：
  1. 导航到其他页面
  2. 返回设置页
  3. `browser_snapshot` 获取对应分组的表单值
- **断言**：
  - 之前保存的值正确回显
  - 不是默认值

### TC-SET-06：恢复默认

- **前置**：设置值已被修改过（非默认值）
- **步骤**：
  1. 点击"恢复默认"按钮
  2. `browser_snapshot` 确认确认弹层或直接执行
  3. `browser_snapshot` 获取最终状态
- **断言**：
  - 恢复后表单值回到默认值
  - 恢复成功反馈

### TC-SET-07：MetaTube 诊断

- **前置**：在设置页，切换到 MetaTube 分组
- **步骤**：
  1. 切换到 MetaTube 分组
  2. 点击诊断按钮
  3. `browser_wait_for` 等待诊断完成
  4. `browser_snapshot` 获取状态
- **断言**：
  - 诊断请求发出
  - 诊断结果正确展示（成功 / 连接失败等）

### TC-SET-08：settings.changed 事件回流

- **前置**：在设置页
- **步骤**：
  1. 修改设置并保存
  2. 在另一个分组观察是否有联动更新
  3. 或刷新页面后观察值是否持久化
- **断言**：
  - `settings.changed` 事件触发后持久化值已更新
  - 未提交的草稿（Dirty 状态）被保护，不被覆盖

### 数据播种流程

播种脚本（待实现）：`config/scan/seed-e2e-data.ps1`

执行顺序：
1. 确保 Worker 已启动（`/api/app/bootstrap` 可访问）
2. `PUT /api/settings` → 写入 MetaTubeConfig.ServerUrl
3. `POST /api/libraries` → 创建 2 个库（scanPaths 指向 `input/lib-a`、`input/lib-b`）
4. `POST /api/libraries/{id}/scan` → 触发扫描（导入假文件为影片记录）
5. `POST /api/libraries/{id}/scrape` → 触发 MetaTube 在线抓取（仅库 A 的已验证 VID）
6. `POST /api/videos/{id}/favorite` → 收藏 2 部影片
