# Task Detail Dialog Spec

## 页面目的

- `Task Detail Dialog` 是任务失败详情与重试弹层。
- 它负责展示任务状态、关键时间线、失败原因和重试入口。

## 页面范围

- 本弹层负责：
  - 展示任务状态胶囊
  - 展示任务关键字段
  - 展示失败原因
  - 提供重试入口
- 本弹层不负责：
  - 展示完整任务中心
  - 替代库管理页内联进度
  - 展示库配置编辑表单

## 数据来源

- 任务详情：
  - `GET /api/tasks/{taskId}`
- 重试任务：
  - `POST /api/tasks/{taskId}/retry`

## 布局

- 使用标准模态弹层结构。
- 顶部保留标题和关闭按钮。
- 主体分为：
  - 状态胶囊区
  - 任务详情信息块
  - 失败原因块
- 底部按钮顺序固定：
  - 左侧 `关闭`
  - 右侧主按钮 `重试任务`

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 固定文案 | 文案为 `任务详情` |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 状态胶囊区 | 状态行 | 是 | 无 | 任务详情 | 展示状态、类型、阶段、进度 |
| 任务信息块 | 信息区 | 是 | 无 | 任务详情 | 展示任务 ID、关联库、时间线 |
| 失败原因区 | 提示块 | 条件 | 无 | 任务详情 | 失败态必显 |
| 关闭按钮 | 次按钮 | 是 | 关闭弹层 | 无 | 不提交 |
| 重试按钮 | 主按钮 | 条件 | 重试任务 | `POST /api/tasks/{taskId}/retry` | 仅失败且可重试时显示 |

## 交互规则

- 从库管理页内联任务、全局活动条或 Home 摘要中的失败任务入口进入。
- 弹层优先展示失败原因，不将用户再跳转到独立任务中心页。
- 点击 `重试任务` 后触发重试接口，并在 pending 态禁用重复点击。

## 状态定义

### Normal

- 可查看任务基础信息与状态。

### Failed

- 显示失败原因与重试入口。

### Retry Pending

- 重试进行中时禁用按钮并更新文案。

## 性能与体验约束

- 任务详情只展示关键字段，不做完整日志查看器。
- 当前任务反馈形态仍以 `库页内联 + 全局活动条 + Home 摘要` 为主，本弹层只承担失败详情与重试。

## 回归点

- 失败任务能打开详情弹层。
- 状态、时间线和失败原因显示正确。
- 可重试任务显示 `重试任务` 按钮。
- 重试成功后能回流到正常任务反馈链路。
