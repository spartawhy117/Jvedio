# Confirm Dialog Spec

## 页面目的

- `Confirm Dialog` 是当前 exe 的通用确认对话框骨架。
- 它负责为删除确认、退出确认、覆盖确认等短路径动作提供统一模态承载。
- 调用方只传入标题与正文文本，不再重复定义底部按钮结构。

## 页面范围

- 本弹层负责：
  - 展示确认标题
  - 展示外部传入的正文文本
  - 提供固定的 `取消 / 确认` 动作
- 本弹层不负责：
  - 复杂表单录入
  - 多步骤流程
  - 上下文菜单型动作集合
  - 页面级错误详情承载

## 归属页面

- `library-management-page`
- 其它后续存在确认动作的页面

## 数据来源

- 标题文案：
  - 由调用方传入
- 正文文本：
  - 由调用方传入
- 确认动作：
  - 由具体业务场景决定

## 布局

- 使用标准白色背景模态对话框。
- 顶部保留标题和关闭按钮。
- 中部正文区为外部传入文本，可展示单段说明或 1 到 2 行摘要。
- 底部按钮顺序固定为：
  - 左侧 `取消`
  - 右侧 `确认`
- 危险场景允许把 `确认` 渲染为红色危险按钮，但不改变按钮文案和顺序。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 调用方传入 | 表达当前确认动作 |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 正文文本区 | 文本块 | 是 | 无 | 调用方传入 | 承载确认说明或摘要 |
| 取消按钮 | 次按钮 | 是 | 关闭弹层 | 无 | 不提交 |
| 确认按钮 | 主按钮 / 危险按钮 | 是 | 执行确认动作 | 具体业务 | 文案固定为 `确认` |

## 交互规则

- 调用方打开弹层时，必须传入当前确认标题和正文文本。
- 点击右上角关闭按钮或 `取消` 时，直接关闭弹层，不提交任何动作。
- 点击 `确认` 后，先关闭或进入 pending 由业务场景决定，但底部结构保持不变。
- 同一时刻只允许一个模态确认对话框处于打开状态。
- 正文文本优先保持短句，不在此组件中扩展成长表格或复杂信息块。

## 状态定义

### Closed

- 默认状态，弹层不显示。

### Confirm

- 展示标题、正文文本和固定底部动作，等待用户确认。

### Pending

- 业务确认动作执行中时，可禁用 `确认` 防止重复提交。

### Error

- 若业务场景要求原地提示失败，可在正文区下方追加一行短错误反馈。

## 性能与体验约束

- 统一确认弹层优先复用同一套视觉与按钮顺序，不为单个页面分叉底部动作布局。
- 正文文本区只承载最小确认信息，避免做成配置表单。
- 危险语义优先通过标题、正文说明和 `确认` 按钮颜色表达，不新增额外第三按钮。

## 回归点

- 不同页面都能复用同一套 `取消 / 确认` 底部结构。
- 标题与正文文本可由调用方正确传入。
- 点击 `取消` 能直接关闭弹层。
- 点击 `确认` 能正确触发对应业务动作。
- 危险场景可把 `确认` 渲染为红色，但文案和顺序保持不变。
