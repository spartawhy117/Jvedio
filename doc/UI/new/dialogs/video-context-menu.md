# Video Context Menu Spec

## 页面目的

- `Video Context Menu` 是影片卡片的统一上下文动作菜单。
- 它负责把右键点击和卡片 `更多` 按钮收口到同一套轻量动作入口。
- 菜单视觉采用纯列表样式，不显示标题栏和分组头。

## 页面范围

- 本菜单负责：
  - 播放当前影片
  - 进入影片详情页
  - 收藏 / 取消收藏
  - 刷新元数据
  - 重新抓取元数据
  - 打开所在文件夹
  - 删除影片原片
- 本菜单不负责：
  - 编辑影片信息
  - 删除资料记录
  - 标签管理
  - 旧 WPF 的高级菜单、复制/剪切文件、路径子菜单

## 数据来源

- 当前影片卡片数据：
  - 当前结果集中的单条影片摘要
- 播放动作：
  - 现有播放调用链路
- 详情动作：
  - 影片详情路由
- 收藏动作：
  - 收藏状态写回接口
- 刷新 / 重新抓取动作：
  - 元数据刷新与重抓任务接口
- 打开文件夹动作：
  - Electron shell 打开本地路径能力
- 删除影片原片动作：
  - 影片删除链路
  - sidecar 与子目录清理
  - 演员关联信息回刷

## 布局

- 使用轻量浮层菜单，不使用模态对话框。
- 菜单锚定在当前影片卡片右侧或 `更多` 按钮附近。
- 菜单不显示顶部标题，不显示说明文案，只保留动作列表。
- 菜单项顺序固定为：
  - `播放`
  - `查看详情`
  - `收藏 / 取消收藏`
  - `刷新元数据`
  - `重新抓取元数据`
  - `打开所在文件夹`
  - `删除影片原片`
- `删除影片原片` 使用红色危险文本样式。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 菜单容器 | 浮层面板 | 条件 | 承载动作列表 | 当前交互态 | 点击外部关闭 |
| 播放 | 菜单项 | 是 | 触发播放 | 当前影片 | 不离开当前结果页 |
| 查看详情 | 菜单项 | 是 | 进入详情页 | 当前影片 | 保留来源页上下文 |
| 收藏 / 取消收藏 | 菜单项 | 是 | 切换收藏状态 | 当前影片收藏态 | 文案跟随状态切换 |
| 刷新元数据 | 菜单项 | 是 | 刷新当前影片元数据 | 当前影片 | 触发状态回刷 |
| 重新抓取元数据 | 菜单项 | 是 | 重新发起抓取 | 当前影片 | 触发状态回刷 |
| 打开所在文件夹 | 菜单项 | 是 | 打开影片目录 | 当前影片路径 | 调用系统外部能力 |
| 删除影片原片 | 危险菜单项 | 是 | 删除影片目录内容并回刷关联数据 | 当前影片路径 / sidecar / 演员索引 | 红色文本，无二次确认 |

## 交互规则

- 右键点击影片卡片，或点击卡片 `更多` 按钮，打开同一套菜单。
- 同一时刻只允许一个影片上下文菜单处于打开状态。
- 点击任意菜单项后立即关闭菜单，再执行对应动作。
- `查看详情` 进入详情页时，必须保留来源页的筛选、排序和分页上下文。
- `收藏 / 取消收藏` 成功后，需要即时更新当前卡片和当前结果集状态。
- `刷新元数据` 与 `重新抓取元数据` 触发后，需要把任务状态回刷到当前页行内摘要或活动条。
- `打开所在文件夹` 属于外部动作，不改变当前页面内容区状态。
- `删除影片原片` 不做二次确认，点击后直接执行删除链路。
- `删除影片原片` 的删除范围固定为：
  - 当前影片文件
  - 当前影片 `.nfo`
  - 同目录下该影片的 poster、thumb、fanart 等 sidecar 资源
  - 当前影片专属子目录及其内部内容
- 删除完成后需要同步更新：
  - 当前结果集移除该影片
  - 演员关联影片计数与最近作品摘要
  - 相关缓存或索引状态

## 状态定义

### Closed

- 默认状态，菜单不显示。

### Open

- 展开菜单并显示 7 个固定动作。

### Favorite On

- 当前影片已收藏，菜单文案显示 `取消收藏`。

### Favorite Off

- 当前影片未收藏，菜单文案显示 `收藏`。

### Task Pending

- 当前影片已有刷新或重抓任务进行中时，对应动作可禁用或展示 pending 反馈。

### Delete Pending

- 删除影片原片执行中时，菜单关闭并由列表区承接状态刷新。

## 性能与体验约束

- 所有影片结果页优先复用同一套上下文菜单规则，不再为不同页面分叉动作集合。
- 第一轮只保留这 7 项，不新增二级菜单和更多低频动作。
- 菜单打开与关闭应保持轻量，不引入额外页面跳闪。
- 菜单项文案保持短句，避免在浮层里加入说明文本。
- 危险项只通过红色文本强调，不再额外弹出确认框。

## 回归点

- 右键点击影片卡片能打开菜单。
- 卡片 `更多` 按钮能打开同一套菜单。
- 菜单项顺序固定且与规格一致。
- `收藏 / 取消收藏` 能即时更新状态。
- `刷新元数据` 与 `重新抓取元数据` 能触发任务反馈回刷。
- `查看详情` 进入详情页后，返回能恢复原结果页上下文。
- `打开所在文件夹` 能正常调用系统打开目录。
- `删除影片原片` 会删除影片、sidecar、专属子目录，并回刷演员关联信息。
