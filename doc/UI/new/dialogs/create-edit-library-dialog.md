# Create Edit Library Dialog Spec

## 页面目的

- `Create Edit Library Dialog` 是媒体库新建与编辑共用的弹层。
- 它负责在同一套字段结构下完成建库与改库。

## 页面范围

- 本弹层负责：
  - 输入媒体库名称
  - 输入扫描目录列表
  - 展示行内校验或接口错误
  - 提交创建或保存修改
- 本弹层不负责：
  - 展示库内影片
  - 展示扫描任务进度
  - 高级扫描策略配置

## 数据来源

- 新建媒体库：
  - `POST /api/libraries`
- 编辑媒体库：
  - `PUT /api/libraries/{libraryId}`

## 布局

- 使用标准模态弹层结构。
- 顶部保留标题和关闭按钮。
- 主体只保留两组字段：
  - `媒体库名称`
  - `扫描目录`
- `扫描目录` 使用路径列表组件，而不是多行纯文本框。
- 底部按钮顺序固定：
  - 左侧 `取消`
  - 右侧主按钮 `保存媒体库`

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 当前模式 | 建库/编辑共用 |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 媒体库名称 | 输入框 | 是 | 编辑库名 | 当前表单态 | 必填 |
| 扫描目录列表 | 路径列表组件 | 是 | 编辑扫描目录列表 | 当前表单态 | 最少 1 条，最多 3 条 |
| 路径输入框 | 行内输入框 | 是 | 编辑单条目录路径 | 当前表单态 | 允许手输或回填 |
| 选择文件夹 | 行内按钮 | 是 | 打开目录选择器 | 本地系统能力 | 选中后回填当前行 |
| 删除目录 | 行内按钮 | 条件 | 删除当前目录行 | 当前表单态 | 仅第 2-3 行显示 |
| 新增目录 | 次按钮 | 条件 | 新增空目录行 | 当前表单态 | 达到 3 条后置灰 |
| 错误反馈区 | 提示条 | 条件 | 展示错误 | 本地校验 / 接口返回 | 无错误时隐藏 |
| 取消按钮 | 次按钮 | 是 | 关闭弹层 | 无 | 不提交 |
| 保存按钮 | 主按钮 | 是 | 提交表单 | 创建 / 更新接口 | pending 时禁用 |

## 交互规则

- 从 `library-management-page` 的 `新建媒体库` 或 `编辑` 进入。
- 编辑模式与新建模式共用同一弹层结构，只是预填已有值。
- 扫描目录按“每行一条路径”展示，不使用手动换行作为主交互。
- 默认至少显示 1 条目录行。
- 点击 `新增目录` 后，在列表底部新增一条空目录行。
- 目录行达到 3 条后，`新增目录` 置灰并提示已达上限。
- 每条目录都可通过 `选择文件夹` 打开系统目录选择器回填。
- 第 2 条和第 3 条目录允许删除；当只剩 1 条时不再允许删除最后一条。
- 如用户在输入框内粘贴多行路径，可按换行自动拆分到多条目录行，但这只是增强能力，不作为主交互。
- 保存前应执行目录去重、空值校验和上限校验。
- 保存失败时，错误反馈在字段下方或底部提示区内联展示。
- 弹层关闭后，不应污染列表页布局。

## 状态定义

### Create

- 新建模式下库名默认为空，目录列表默认只有 1 条空行。

### Edit

- 编辑模式下带入当前库名和 1-3 条已有目录。

### Pending

- 提交中时禁用重复点击。

### Error

- 校验失败或接口失败时显示错误反馈区。

### Max Reached

- 目录条数达到 3 条时，`新增目录` 不再可用。

## 性能与体验约束

- 弹层字段保持最小化，不在第一轮加入更多配置项。
- 建库与编辑必须共用一套弹层壳，避免分叉两种 UI。
- 目录列表组件应优先服务“少量固定条目”的录入场景，不引入复杂拖拽排序。
- 目录数量上限固定为 3，避免弹层纵向失控。

## 回归点

- `新建媒体库` 能打开弹层。
- `编辑` 能复用同一弹层并预填值。
- 扫描目录以路径列表组件展示，而不是多行文本框。
- `新增目录` 最多只能增加到 3 条。
- `选择文件夹` 能回填当前目录行。
- 删除目录行后，剩余目录值保持正确。
- 保存成功后列表页能刷新。
- 保存失败时有明确错误反馈。
