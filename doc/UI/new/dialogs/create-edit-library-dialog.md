# Create Edit Library Dialog Spec

## 页面目的

- `Create Edit Library Dialog` 是媒体库新建与编辑共用的弹层。
- 它负责在同一套字段结构下完成建库与改库。

## 页面范围

- 本弹层负责：
  - 输入媒体库名称
  - 输入扫描目录
  - 展示行内校验或接口错误
  - 提交创建或保存修改
- 本弹层不负责：
  - 展示库内影片
  - 展示扫描任务进度
  - 高级扫描策略配置

## 数据来源

- 新建媒体库：
  - `POST /api/libraries`
- 编辑媒体库：
  - `PUT /api/libraries/{libraryId}`

## 布局

- 使用标准模态弹层结构。
- 顶部保留标题和关闭按钮。
- 主体只保留两组字段：
  - `媒体库名称`
  - `扫描目录`
- 底部按钮顺序固定：
  - 左侧 `取消`
  - 右侧主按钮 `保存媒体库`

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 当前模式 | 建库/编辑共用 |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 媒体库名称 | 输入框 | 是 | 编辑库名 | 当前表单态 | 必填 |
| 扫描目录 | 多行输入框 | 是 | 编辑扫描目录列表 | 当前表单态 | 支持多行路径 |
| 错误反馈区 | 提示条 | 条件 | 展示错误 | 本地校验 / 接口返回 | 无错误时隐藏 |
| 取消按钮 | 次按钮 | 是 | 关闭弹层 | 无 | 不提交 |
| 保存按钮 | 主按钮 | 是 | 提交表单 | 创建 / 更新接口 | pending 时禁用 |

## 交互规则

- 从 `library-management-page` 的 `新建媒体库` 或 `编辑` 进入。
- 编辑模式与新建模式共用同一弹层结构，只是预填已有值。
- 保存失败时，错误反馈在字段下方或底部提示区内联展示。
- 弹层关闭后，不应污染列表页布局。

## 状态定义

### Create

- 新建模式下字段默认为空或预填建议值。

### Edit

- 编辑模式下带入当前库名和扫描目录。

### Pending

- 提交中时禁用重复点击。

### Error

- 校验失败或接口失败时显示错误反馈区。

## 性能与体验约束

- 弹层字段保持最小化，不在第一轮加入更多配置项。
- 建库与编辑必须共用一套弹层壳，避免分叉两种 UI。

## 回归点

- `新建媒体库` 能打开弹层。
- `编辑` 能复用同一弹层并预填值。
- 保存成功后列表页能刷新。
- 保存失败时有明确错误反馈。
