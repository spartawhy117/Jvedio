# Create Edit Library Dialog Spec

## 页面目的

- `Create Edit Library Dialog` 是媒体库新建与编辑共用的弹层。
- 它负责在同一套字段结构下完成建库与改库。

## 页面范围

- 本弹层负责：
  - 输入媒体库名称
  - 输入扫描目录列表
  - 展示行内校验或接口错误
  - 提交创建或保存修改
- 本弹层不负责：
  - 展示库内影片
  - 展示扫描任务进度
  - 高级扫描策略配置

## 数据来源

- 新建媒体库：
  - `POST /api/libraries`
- 编辑媒体库：
  - `PUT /api/libraries/{libraryId}`

## 布局

- 使用白色背景的标准模态弹层结构。
- 顶部保留标题和关闭按钮。
- 主体只保留两组字段：
  - `媒体库名称`
  - `扫描目录`
- `扫描目录` 使用 3 个固定路径输入框，不做动态增删。
- 底部按钮顺序固定：
  - 左侧 `取消`
  - 右侧主按钮 `保存媒体库`

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 当前模式 | 建库/编辑共用 |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 媒体库名称 | 输入框 | 是 | 编辑库名 | 当前表单态 | 必填 |
| 目录 1 | 路径输入框 | 是 | 编辑第 1 条目录路径 | 当前表单态 | 必填 |
| 目录 2 | 路径输入框 | 是 | 编辑第 2 条目录路径 | 当前表单态 | 可留空 |
| 目录 3 | 路径输入框 | 是 | 编辑第 3 条目录路径 | 当前表单态 | 可留空 |
| 选择文件夹 | 行内按钮 | 是 | 打开目录选择器 | 本地系统能力 | 选中后回填当前行 |

## 交互规则

- 从 `library-management-page` 的 `新建媒体库` 或 `编辑` 进入。
- 编辑模式与新建模式共用同一弹层结构，只是预填已有值。
- 扫描目录固定展示 3 条输入框，不提供新增目录或删除目录的动态交互。
- 每条目录都可通过对应的 `选择` 按钮打开系统目录选择器回填。
- 系统目录选择器返回后，只回填当前行，不自动改写其它目录行。
- 保存时按顺序读取 3 个输入框：
  - 第 1 条不能为空
  - 第 2 条和第 3 条可为空
- 保存前应执行：
  - 去除首尾空白
  - 空值过滤
  - 重复路径校验
  - 每条非空路径的可访问性校验
- 保存失败时，错误反馈在字段下方或底部提示区内联展示。
- 保存成功后立即关闭弹层，并由列表页承接刷新结果。
- 弹层关闭后，不应污染列表页布局。

## 状态定义

### Create

- 新建模式下库名默认为空，3 条目录输入框均为空。

### Edit

- 编辑模式下带入当前库名和已有目录，并按顺序回填到 3 条固定输入框。

### Pending

- 提交中时禁用重复点击。

### Error

- 校验失败或接口失败时显示错误反馈区。

## 性能与体验约束

- 弹层字段保持最小化，不在第一轮加入更多配置项。
- 建库与编辑必须共用一套弹层壳，避免分叉两种 UI。
- 目录数量固定为 3 条，优先降低交互成本，而不是追求弹性配置。
- 目录区不引入新增、删除、排序等额外交互。

## 回归点

- `新建媒体库` 能打开弹层。
- `编辑` 能复用同一弹层并预填值。
- 扫描目录以 3 个固定输入框展示，而不是多行文本框或动态列表。
- `选择文件夹` 能回填对应目录行。
- 保存时会校验非空路径是否可访问。
- 重复路径或不可访问路径会给出明确错误反馈。
- 保存成功后列表页能刷新。
- 保存失败时有明确错误反馈。
