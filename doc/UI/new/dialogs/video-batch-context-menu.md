# Video Batch Context Menu Spec

## 页面目的

- `Video Batch Context Menu` 是多选影片后的统一批量动作菜单。
- 它负责把“勾选多张影片卡后右键”的入口收口到同一套轻量动作菜单。
- 菜单视觉延续 `video-context-menu` 的白色轻量浮层列表风格，不使用标题栏和分组头。

## 页面范围

- 本菜单负责：
  - 展示当前已选数量
  - 批量加入收藏
  - 批量重新抓取元数据
  - 取消当前选择
  - 批量删除影片原片
- 本菜单不负责：
  - 单张影片详情跳转
  - 单张影片播放
  - 顶部批量工具条
  - 旧 WPF 的高级批处理子菜单

## 归属页面

- `library-page`
- `favorites-page`
- `actor-detail-page`

## 数据来源

- 当前选中影片集合：
  - `GET /api/libraries/{libraryId}/videos`
  - `GET /api/videos/favorites`
  - `GET /api/actors/{actorId}/videos`
- 收藏动作：
  - 收藏状态写回接口
- 重新抓取动作：
  - 元数据重抓任务接口
- 删除影片原片动作：
  - 影片删除链路
  - sidecar 与子目录清理
  - 关联结果集回刷

## 布局

- 使用轻量浮层菜单，不使用模态对话框。
- 菜单锚定在当前选中卡片区域的右键位置附近。
- 菜单顶部保留一行只读的已选数量提示，例如 `已选择 12 项`。
- 菜单项顺序固定为：
  - `加入收藏`
  - `重新抓取元数据`
  - `取消选择`
  - `删除影片原片`
- `删除影片原片` 使用红色危险文本样式。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 菜单容器 | 浮层面板 | 条件 | 承载批量动作列表 | 当前交互态 | 点击外部关闭 |
| 已选数量 | 只读提示行 | 是 | 无 | 当前选择态 | 仅表达已选数量，不可点击 |
| 加入收藏 | 菜单项 | 是 | 批量加入收藏 | 当前选中影片集合 | 成功后即时回刷当前结果集 |
| 重新抓取元数据 | 菜单项 | 是 | 批量发起重抓任务 | 当前选中影片集合 | 触发任务状态回刷 |
| 取消选择 | 菜单项 | 是 | 清空当前勾选态 | 当前选择态 | 不跳转页面 |
| 删除影片原片 | 危险菜单项 | 是 | 批量删除影片目录内容并回刷关联数据 | 当前影片路径 / sidecar / 演员索引 | 红色文本，无额外标题 |

## 交互规则

- 用户至少勾选 1 张影片卡后，页面进入多选态。
- 右键当前选中卡片或选中区域时，打开同一套批量菜单。
- 如果右键的是未选中卡片，则优先走单张 `video-context-menu`，而不是批量菜单。
- 同一时刻只允许一个批量菜单处于打开状态。
- 点击任意菜单项后立即关闭菜单，再执行对应动作。
- `加入收藏` 成功后，需要即时更新当前结果集中的收藏状态。
- `重新抓取元数据` 触发后，需要把任务状态回刷到当前结果页摘要或任务反馈区域。
- `取消选择` 执行后，应立即清空当前勾选态并退出多选状态。
- `删除影片原片` 不做二次确认，点击后直接执行删除链路。
- `删除影片原片` 的删除范围固定为：
  - 当前选中影片文件
  - 当前选中影片 `.nfo`
  - 同目录下对应影片的 poster、thumb、fanart 等 sidecar 资源
  - 当前选中影片各自专属子目录及其内部内容
- 删除完成后需要同步更新：
  - 当前结果集移除对应影片
  - 演员关联影片计数与结果摘要
  - 相关缓存或索引状态

## 状态定义

### Closed

- 默认状态，菜单不显示。

### Open

- 展开菜单并显示已选数量与 4 个固定动作。

### Favorite Pending

- 批量加入收藏执行中时，菜单关闭并由结果区承接状态回刷。

### Task Pending

- 批量重抓执行中时，菜单关闭并由任务反馈承接状态显示。

### Delete Pending

- 批量删除执行中时，菜单关闭并由结果区承接状态刷新。

## 性能与体验约束

- 多选批量动作统一收口到同一套菜单，不再为不同页面分叉动作集合。
- 当前版本不再维护结果区顶部批量工具条，避免与右键菜单重复。
- 菜单项文案保持短句，不增加说明段或复杂分组。
- 菜单打开与关闭应保持轻量，不引入额外页面跳闪。
- 危险项只通过红色文本强调，不额外引入二次确认弹层。

## 回归点

- 勾选多张影片卡后，右键能打开批量菜单。
- 菜单顶部能正确显示已选数量。
- `library-page`、`favorites-page`、`actor-detail-page` 都复用同一套批量菜单项。
- `加入收藏` 能即时更新当前结果集状态。
- `重新抓取元数据` 能触发任务反馈回刷。
- `取消选择` 能立即清空当前勾选态。
- `删除影片原片` 会删除影片、sidecar、专属子目录，并回刷关联数据。
