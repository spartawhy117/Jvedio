# Shared UI Components Spec

## 页面目的

- 统一当前 exe 各页面重复出现的基础 UI 组件。
- 页面规格文档不再各自重复解释相同组件，只引用本文件。

## 适用范围

- `库管理`
- `库内容`
- `喜欢`
- `演员`
- `演员详情`
- `影片详情`
- `设置`
- 各类弹层

## 通用文档规则

- 线框图负责：
  - 空间布局
  - 信息层级
  - 主要视觉关系
- 页面 `.md` 负责：
  - 元素功能
  - 数据来源
  - 状态定义
  - 交互规则
  - 性能与回归点

## 统一影片卡片

### 目的

- 为 `库内容`、`喜欢`、`演员详情关联影片` 提供统一的影片结果项结构。

### 组成

- 大图主视觉区
- 右上角勾选按钮
- 图底细进度 / 状态条
- 居中主标题
- 居中次级元数据行
- 悬浮或右键动作入口

### 规则

- 结构参考 Jellyfin 的 `backdrop` 影片卡，优先使用宽图主视觉，而不是左图右文的资料块。
- 图像区占卡片绝大多数高度，主视觉优先让给影片海报或主图展示。
- 卡片下半文字区尽量压缩，只保留标题和年月两行，不再扩展说明块。
- 卡片右上角保留勾选按钮，未勾选时只表示可选；用户勾选后表示进入批量操作选择态。
- 主标题优先用 `VID + 标题短名` 的合并写法，避免再拆多行身份信息。
- 次级元数据只保留一行年月信息，例如 `2026.03`，不再同时堆厂牌和状态。
- 播放进度、已整理或已刮削等状态优先通过主图底边细条表达，不扩展成大 badge 区。
- 点击卡片或主链接进入 `video-detail-page`。
- 右键或 `更多` 按钮统一打开 `dialogs/video-context-menu.md` 定义的动作菜单。
- 当已勾选至少 1 张影片卡后，右键当前选中卡片或选中区域，统一打开 `dialogs/video-batch-context-menu.md` 定义的批量菜单。

## 统一演员卡片

### 目的

- 为 `演员` 聚合页以及后续任何演员结果网格提供统一的人像卡片结构。

### 组成

- 竖向头像主视觉区
- 居中演员名
- 居中次级信息行

### 规则

- 结构参考 Jellyfin 的 `person card`，优先保留大头像和居中文本，不退回资料面板式卡片。
- 头像区保持固定竖向比例，窗口伸缩时不改成横图或自由裁切块。
- 次级信息行默认只保留一行，优先展示 `作品数`，必要时才替换为别名或角色说明。
- 演员卡默认不承载复杂 badge、长简介或多行操作区。
- 点击卡片进入 `actor-detail-page`。

## 结果摘要条

### 目的

- 在聚合结果页顶部用短标签快速表达当前结果集状态。

### 组成

- 总数
- 缺资源数量
- 当前筛选状态

### 规则

- 只表达当前结果集状态，不重复页面标题。
- 采用短词和数字，不写长句说明。
- 当结果页支持返回链路恢复时，摘要条可承载当前筛选与当前选择对象的短说明。

## 结果页查询工具栏

### 目的

- 为 `库内容`、`喜欢`、`演员` 提供统一的查询、排序与刷新入口。

### 组成

- 关键字搜索框
- 排序切换组
- 刷新按钮
- 条件态短标签

### 规则

- 工具栏保持单行优先，桌面宽度足够时不拆成多排杂糅布局。
- 搜索框始终放在左侧，承担当前页面最核心的关键字筛选输入。
- 排序切换组位于搜索框右侧，优先用 2 到 3 个轻量按钮表达常用排序，不扩展成复杂下拉面板。
- 刷新按钮放在最右侧，语义固定为重新拉取当前结果集，而不是重置所有查询条件。
- 当前已生效的筛选条件可用短标签贴在工具栏中部或右侧，不重复展开成长说明句。
- 轻量聚合页如 `喜欢` 可裁掉次要筛选项，但仍复用同一工具栏骨架。

## 行内 Action Strip

### 目的

- 把单条记录上的多个操作收口到同一行，避免顶部按钮区和行内按钮区重复。

### 适用对象

- 媒体库行
- 任务行
- 其它单条记录管理项

### 规则

- 所有按钮保持在同一水平线上。
- 默认线框示例固定为：
  - `打开`
  - `扫描`
  - `编辑`
  - `删除`
- 动作顺序遵循：
  - 主浏览动作
  - 主执行动作
  - 编辑动作
  - 删除动作
- 样式约定：
  - 主浏览动作：次按钮或链接按钮
  - 主执行动作：主按钮
  - 编辑动作：次按钮
  - 删除动作：危险按钮

## 状态 Badge

### 目的

- 用短词而不是长句表达当前状态。

### 规则

- 一个元素只表达一个状态。
- 状态词优先固定枚举，不在页面里自由发挥。
- 颜色语义固定：
  - 绿色：待执行 / 可处理
  - 蓝色：进行中
  - 灰色：稳定 / 已同步
  - 红色：失败 / 风险

## 结果状态组件

### 目的

- 为聚合结果页和弹层内结果区统一 `Loading / Empty / Error / Refresh` 的反馈表达，避免每页各自拼装。

### 组成

- loading 骨架或占位块
- empty 文案区
- error 文案区
- 刷新 / 重试按钮

### 规则

- `Loading` 优先使用骨架块或占位条，不直接堆叠长句提示。
- `Empty` 只表达当前结果为空及最小引导，例如 `暂无结果` 与 `清空筛选后再试`。
- `Error` 必须同时提供短错误说明与 `刷新 / 重试` 入口，不要求在共享组件里承载完整异常详情。
- 结果状态组件默认居中放在结果区主内容位置，不抢占页面标题与工具栏区域。
- 同一结果区一次只显示一种主状态，不叠加多个提示层。

## 分页控件

### 目的

- 为结果集页提供统一翻页结构。

### 规则

- 默认由：
  - 上一页
  - 当前页文本框
  - 下一页
  - 跳转执行按钮
  组成。
- 放置在结果区右下角。
- 分页控件采用单行布局，顺序固定为：
  - `<`
  - `当前页文本框`
  - `>`
  - `跳转`
- 中间文本框默认显示 `当前页 / 总页数`，例如 `1 / 24`。
- 用户点击中间文本框后，可直接输入目标页码。
- `跳转` 动作读取中间文本框的输入页码，先校验是否合法，再直接切换到目标页并刷新当前结果集。
- 跳转完成后，中间文本框恢复为新的 `当前页 / 总页数` 显示，例如 `12 / 24`。
- 当输入页码小于 `1` 或大于总页数时，应阻止提交并保留当前页状态。
- 分页状态应尽量写入当前路由或返回态，保证从详情页返回后恢复原上下文。

## 返回链路状态

### 目的

- 为 `库内容`、`喜欢`、`演员`、`演员详情`、`影片详情` 提供统一的“来源页恢复”语义。

### 规则

- 结果页负责持有自己的筛选、排序和分页状态。
- 详情页优先通过 `backTo` 或等效返回态恢复来源页，而不是硬编码返回某个固定页面。
- 若存在二级详情关系，例如 `Actors -> Actor Detail -> Video Detail`，返回链路应先回二级详情，再回结果页。

## 通用弹窗

### 目的

- 承担新建、编辑、删除确认、失败详情这类局部动作，不强制跳完整页面。

### 规则

- 同类动作优先复用同一个弹窗壳。
- `create/edit` 优先共用一套字段结构。
- 关闭动作始终存在：
  - 右上角关闭
  - 底部取消或关闭
- 底部按钮顺序：
  - 左侧次按钮
  - 右侧确认或主动作按钮

## 列表页信息密度规则

- 列表页优先展示：
  - 身份信息
  - 数量信息
  - 最近状态
  - 行动入口
- 列表页避免展示：
  - 长路径全文
  - 多行配置细节
  - 只有进入编辑态才需要的信息

## 状态说明文本

- 列表中的次级说明文本只保留一行短句。
- 用来补充 badge 无法表达的上下文。
- 不用于堆配置项。

## 文档模板建议

- 每个页面规格至少包含：
  - `页面目的`
  - `页面范围`
  - `数据来源`
  - `元素清单`
  - `交互规则`
  - `状态定义`
  - `性能与体验约束`
  - `回归点`

## 当前特别约束

- 没有真实实现的功能，不要只因为旧设计稿里出现过就继续画成可点击入口。
- 如果某功能当前不存在稳定产品链路：
  - 可以在文档里说明原因
  - 但不应继续占据页面主操作位
