# Shared UI Components Spec

## 页面目的

- 统一当前 exe 各页面重复出现的基础 UI 组件。
- 页面规格文档不再各自重复解释相同组件，只引用本文件。

## 适用范围

- `库管理`
- `库内容`
- `喜欢`
- `演员`
- `演员详情`
- `类别`
- `系列`
- `影片详情`
- `设置`
- 各类弹层

## 通用文档规则

- 线框图负责：
  - 空间布局
  - 信息层级
  - 主要视觉关系
- 页面 `.md` 负责：
  - 元素功能
  - 数据来源
  - 状态定义
  - 交互规则
  - 性能与回归点

## 统一影片卡片

### 目的

- 为 `库内容`、`喜欢`、`类别`、`系列`、`演员详情关联影片` 提供统一的影片结果项结构。

### 组成

- 主图占位
- VID
- 标题或副标题
- 最小元数据行
- 结果页动作入口

### 规则

- 卡片首屏优先表达影片身份，不堆更多描述。
- 资源状态、路径和附加说明只在必要时补一行短文本。
- 点击卡片或主链接进入 `video-detail-page`。

## 结果摘要条

### 目的

- 在聚合结果页顶部用短标签快速表达当前结果集状态。

### 组成

- 总数
- 缺资源数量
- 当前筛选状态

### 规则

- 只表达当前结果集状态，不重复页面标题。
- 采用短词和数字，不写长句说明。

## 行内 Action Strip

### 目的

- 把单条记录上的多个操作收口到同一行，避免顶部按钮区和行内按钮区重复。

### 适用对象

- 媒体库行
- 任务行
- 其它单条记录管理项

### 规则

- 所有按钮保持在同一水平线上。
- 动作顺序遵循：
  - 主浏览动作
  - 主执行动作
  - 编辑动作
  - 删除动作
- 样式约定：
  - 主浏览动作：次按钮或链接按钮
  - 主执行动作：主按钮
  - 编辑动作：次按钮
  - 删除动作：危险按钮

## 状态 Badge

### 目的

- 用短词而不是长句表达当前状态。

### 规则

- 一个元素只表达一个状态。
- 状态词优先固定枚举，不在页面里自由发挥。
- 颜色语义固定：
  - 绿色：待执行 / 可处理
  - 蓝色：进行中
  - 灰色：稳定 / 已同步
  - 红色：失败 / 风险

## 分页控件

### 目的

- 为结果集页提供统一翻页结构。

### 规则

- 默认由：
  - 上一页
  - 当前页号
  - 下一页
  组成。
- 放置在结果区右下角。
- 当前页号使用轻强调态。

## 通用弹窗

### 目的

- 承担新建、编辑、删除确认、失败详情这类局部动作，不强制跳完整页面。

### 规则

- 同类动作优先复用同一个弹窗壳。
- `create/edit` 优先共用一套字段结构。
- 关闭动作始终存在：
  - 右上角关闭
  - 底部取消或关闭
- 底部按钮顺序：
  - 左侧次按钮
  - 右侧确认或主动作按钮

## 列表页信息密度规则

- 列表页优先展示：
  - 身份信息
  - 数量信息
  - 最近状态
  - 行动入口
- 列表页避免展示：
  - 长路径全文
  - 多行配置细节
  - 只有进入编辑态才需要的信息

## 状态说明文本

- 列表中的次级说明文本只保留一行短句。
- 用来补充 badge 无法表达的上下文。
- 不用于堆配置项。

## 文档模板建议

- 每个页面规格至少包含：
  - `页面目的`
  - `页面范围`
  - `数据来源`
  - `元素清单`
  - `交互规则`
  - `状态定义`
  - `性能与体验约束`
  - `回归点`

## 当前特别约束

- 没有真实实现的功能，不要只因为旧设计稿里出现过就继续画成可点击入口。
- 如果某功能当前不存在稳定产品链路：
  - 可以在文档里说明原因
  - 但不应继续占据页面主操作位
