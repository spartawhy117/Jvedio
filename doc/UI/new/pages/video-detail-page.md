# Video Detail Page Spec

## 页面目的

- `Video Detail Page` 是统一的影片详情页。
- 它负责承接列表页进入后的详情阅读、主播放动作、关联演员浏览和返回来源页。

## 页面范围

- 本页负责：
  - 展示顶部 fanart 主视觉、基础元数据和 sidecar 状态
  - 提供主播放入口
  - 提供“打开文件夹”动作
  - 展示关联演员并进入演员详情页
  - 根据返回链路回到来源页或来源分组
- 本页不负责：
  - 媒体库管理
  - 演员完整资料编辑
  - 影片元数据手工编辑
  - 内嵌播放器

## 数据来源

- 影片详情：
  - `GET /api/videos/{videoId}`
- 详情顶部背景图：
  - 优先 `GET /api/videos/{videoId}/fanart`
  - 缺失时不回退到左侧独立主图，而是保留渐变背景
- 播放调用：
  - `POST /api/videos/{videoId}/play`
- 关联演员：
  - `GET /api/videos/{videoId}` 返回的 `actors`

## 主要能力

- 展示 fanart 顶部主视觉和标题信息
- 展示 sidecar 状态
- 显示首次入库时间
- 触发播放调用
- 贴近文件路径提供“打开文件夹”动作
- 通过统一回退组件恢复来源页
- 进入演员详情页

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 顶部先渲染一整块 `hero`：
  - 底层是 `fanart` 背景图与 scrim 遮罩
  - 前景内容采用同一水平带三段铺开：左侧标题、中间 `2 x 2` sidecar 状态、右侧主播放按钮
  - 元数据卡片网格继续位于同一块 `hero` 背景板内，并与顶部三段内容保持固定间距
- `hero` 下方按垂直顺序继续排布：
  - 横向演员条
  - 文件路径块
  - 外部来源页块
- 顶部回退组件当前只保留箭头按钮，不再展示回退目标文字。
- “打开文件夹”按钮位于文件路径块标题行右侧。
- 窄屏下顶部三段会折叠为单列，但模块顺序保持不变。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 回退组件 | 次按钮 | 条件 | 返回来源页 | 路由历史 | 当前只显示箭头，不显示文本 |
| 页面标题 | 静态标题 | 是 | 无 | `title / displayTitle / vid` | 位于 hero 左侧主信息区 |
| sidecar 状态组 | badge 组 | 是 | 无 | `sidecars` | hero 顶部中段采用 `2 x 2` 形式展示 NFO / Poster / Thumb / Fanart |
| fanart 背景层 | 内容层 | 条件 | 无 | `sidecars.fanart` | 仅作为顶部背景板，不再作为单独前景主图 |
| 主播放按钮 | 主按钮 | 是 | 调用播放接口 | `playback` | 位于 hero 顶部右段，为更大的矩形按钮 |
| CTA 提示 | 辅助文本 | 条件 | 无 | `playback.reason` | 仅在不可播放时显示在按钮下方 |
| 元数据区 | 信息卡网格 | 是 | 无 | 详情接口 | 位于 `hero` 下半区，当前展示 VID、库、厂商、系列、发布日期、首次入库、时长、导演、播放次数、上次播放、上次扫描等 |
| 关联演员区 | 横向卡片条 | 条件 | 进入演员详情 | `actors` | 位于 `hero` 下方，复用通用演员卡，头像在上、名字在下，可横向滚动 |
| 文件路径区 | 信息块 | 是 | 打开文件夹 | `path` | 路径标题行右侧紧邻“打开文件夹”动作 |
| 外部来源页 | 链接 | 条件 | 新窗口打开 | `webUrl` | 无值时隐藏 |

## 交互规则

- 可从 `library-page`、`favorites-page`、`actor-detail-page` 进入本页。
- 回退组件只保留图标点击区域，不在 UI 上渲染任何返回说明文字。
- 从本页进入演员详情时，路由仍带 `Video Detail` 标签，但该标签当前不在返回组件上直接显示。
- 点击 `播放` 后调用播放接口；请求进行中禁用重复点击。
- 当 `playback.canPlay = false` 时，按钮可禁用，并展示原因提示。
- 点击演员条中的演员卡进入演员详情。
- 点击“打开文件夹”后定位到当前影片文件所在目录。

## 状态定义

### Loading

- 首次进入页面时显示详情 loading。

### Error

- 详情拉取失败时显示错误状态，并保留回退组件。

### Empty

- 未找到影片时显示空状态，并保留回退组件。

### Play Pending

- 播放调用进行中时禁用主按钮。

## 性能与体验约束

- 列表卡与详情页图片职责必须分开：
  - 列表卡默认使用 `poster`
  - 详情页顶部背景默认使用 `fanart`
- 详情页不再保留固定左侧前景海报列。
- 顶部标题、sidecar 状态和主播放按钮必须在宽屏下保持同一水平带内均匀铺开。
- 大模块之间必须保留稳定间距，窗口拉伸时只能扩展留白和列宽，不能让演员条、路径块、来源块互相挤压贴连。
- 主 CTA 必须稳定停留在 hero 顶部右段，不能回退到正文区或图片下方。
- 路径信息和目录动作必须保持邻接，避免信息和动作分离。
- 关联演员区必须复用统一演员卡视觉，并以横向条带形式独立出现在 `hero` 下方。
- 当前页面不展示影片简介块。
- 窄屏下顶部三段可折叠为单列，但标题、sidecar、CTA、演员条、路径和来源块顺序不能打乱。

## 回归点

- 从库页、喜欢页、演员详情页进入后，回退入口仍可用，且不再显示长文本。
- fanart 存在时顶部 hero 正确展示背景层；缺失时保留安全渐变背景。
- 播放按钮位于 hero 顶部右段，且不可播放时能显示原因。
- sidecar 状态在 hero 顶部中段以 `2 x 2` 形式展示。
- 关联演员以下方横向条带形式出现，不再嵌在右侧正文列中。
- “打开文件夹”位于文件路径标题行右侧。
- 关联演员以卡片形式显示头像和名字，并可跳转。
- 首次入库时间可显示，格式稳定。
