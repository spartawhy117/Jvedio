# Video Detail Page Spec

## 页面目的

- `Video Detail Page` 是统一的影片详情页。
- 它负责承接列表页进入后的详情阅读、主播放动作、关联演员浏览和返回来源页。

## 页面范围

- 本页负责：
  - 展示影片主图、基础元数据、资源状态和简介
  - 提供主播放入口
  - 提供“打开文件夹”动作
  - 展示关联演员并进入演员详情页
  - 根据返回链路回到来源页或来源分组
- 本页不负责：
  - 媒体库管理
  - 演员完整资料编辑
  - 影片元数据手工编辑
  - 内嵌播放器

## 数据来源

- 影片详情：
  - `GET /api/videos/{videoId}`
- 详情主图：
  - 优先 `GET /api/videos/{videoId}/thumb`
  - 回退 `GET /api/videos/{videoId}/poster`
- 播放调用：
  - `POST /api/videos/{videoId}/play`
- 关联演员：
  - `GET /api/videos/{videoId}` 返回的 `actors`

## 主要能力

- 展示影片详情主信息
- 展示 sidecar 状态
- 显示首次入库时间
- 触发播放调用
- 贴近文件路径提供“打开文件夹”动作
- 通过统一回退组件恢复来源页
- 进入演员详情页

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 内容区采用左右分栏：
  - 左侧为主图、VID、sidecar 状态
  - 右侧为标题、元数据、主 CTA、简介、演员卡、路径信息、来源页链接
- 主播放按钮位于右侧信息流中上部，优先级高于简介和路径。
- “打开文件夹”按钮与“文件路径”同一信息块展示，并贴在标题行右侧。
- 顶部使用统一回退组件，箭头后文案表示“将返回到的页面或分组名称”。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 回退组件 | 次按钮 | 条件 | 返回来源页 | 路由历史 `label` | 显示返回目标名，不显示当前长标题 |
| 页面标题 | 静态标题 | 是 | 无 | `displayTitle / vid` | 作为详情页头部标题 |
| 主图区 | 内容块 | 是 | 无 | `thumb / poster` | 详情页主图优先 thumb，缺失再回退 poster 和占位 |
| VID 标识 | 状态标识 | 是 | 无 | `vid` | 展示当前影片编号 |
| sidecar 状态组 | badge 组 | 是 | 无 | `sidecars` | 展示 NFO / Poster / Thumb / Fanart 状态 |
| 主播放按钮 | 主按钮 | 是 | 调用播放接口 | `playback` | 位于右栏主 CTA 位；不可播放时显示原因 |
| 元数据区 | 信息块 | 是 | 无 | 详情接口 | 至少展示库、发布日期、首次入库时间、时长等 |
| 简介区 | 文本块 | 条件 | 无 | `plot / outline` | 作为正文说明区 |
| 关联演员区 | 卡片网格 | 条件 | 进入演员详情 | `actors` | 复用通用演员卡紧凑样式，至少展示头像和名字 |
| 文件路径区 | 信息块 | 是 | 打开文件夹 | `path` | 路径标题行右侧紧邻“打开文件夹”动作 |
| 外部来源页 | 链接 | 条件 | 新窗口打开 | `webUrl` | 无值时隐藏 |

## 交互规则

- 可从 `library-page`、`favorites-page`、`actor-detail-page` 进入本页。
- 回退组件必须优先读取历史栈里的 `label`，语义固定为“返回目标页面 / 分组名”。
- 从本页进入演员详情时，返回标签应为 `影片详情 / Video Detail`，而不是当前影片标题。
- 点击 `播放` 后调用播放接口；请求进行中禁用重复点击。
- 当 `playback.canPlay = false` 时，按钮可禁用，并展示原因提示。
- 点击演员卡进入演员详情。
- 点击“打开文件夹”后定位到当前影片文件所在目录。

## 状态定义

### Loading

- 首次进入页面时显示详情 loading。

### Error

- 详情拉取失败时显示错误状态，并保留回退组件。

### Empty

- 未找到影片时显示空状态，并保留回退组件。

### Play Pending

- 播放调用进行中时禁用主按钮。

## 性能与体验约束

- 详情页主图与列表卡图源职责必须分开：
  - 列表卡默认使用 `poster`
  - 详情页默认使用 `thumb`
- 右侧主 CTA 在桌面宽度下应具备足够存在感，不允许继续退回左栏图片区下方。
- 路径信息和目录动作必须保持邻接，避免信息和动作分离。
- 关联演员区必须复用统一演员卡视觉，不再单独创建弱化标签样式。
- 窄屏下左右分栏可折叠为单列，但 CTA、路径和演员卡顺序不能打乱。

## 回归点

- 从库页、喜欢页、演员详情页进入后，回退文案和回退目标正确。
- 详情主图优先展示 `thumb`，缺失时正确回退到 `poster` 或占位。
- 播放按钮位于右侧主信息区，且不可播放时能显示原因。
- “打开文件夹”位于文件路径标题行右侧。
- 关联演员以卡片形式显示头像和名字，并可跳转。
- 首次入库时间可显示，格式稳定。
