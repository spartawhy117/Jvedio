# Actor Detail Page Spec

## 页面目的

- `Actor Detail Page` 是演员列表进入后的独立详情页。
- 它负责展示演员主标识信息和关联影片结果集。

## 页面范围

- 本页负责：
  - 展示演员名称与 ID
  - 展示演员关联影片结果集
  - 从关联影片进入影片详情页
  - 提供返回 Actors 列表的链路
- 本页不负责：
  - 演员编辑
  - 媒体库管理
  - 影片详情主内容本身

## 数据来源

- 演员详情：
  - `GET /api/actors/{actorId}`
- 演员关联影片：
  - `GET /api/actors/{actorId}/videos`
- 影片详情：
  - `GET /api/videos/{videoId}`

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 当前线框只表达右侧内容区。
- 内容区采用单栏结果布局：
  - 主结果区顶部左侧以同一行横向排布演员名称与 `Actor ID`
  - 主结果区主体展示关联影片卡片网格
- 页面不再保留独立左侧信息栏，也不再显示 `关联影片` 分区标题。
- 主结果区复用 `../shared/shared-components.md` 中定义的 `统一影片卡片` 与 `分页控件`。
- 关联影片卡片采用 Jellyfin 风格宽图结构：大图主视觉、图底细进度条、居中标题与一行次级元数据。
- 顶部右侧保留返回 Actors 列表按钮。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 页面标题 | 静态标题 | 是 | 无 | 固定文案 | 文案为 `演员详情 / Actor Detail` |
| 返回按钮 | 次按钮 | 是 | 返回 Actors 列表 | 当前返回态 | 恢复原列表状态 |
| 演员名称 / ID | 顶部信息 | 是 | 无 | `GET /api/actors/{actorId}` | 固定放在主结果区顶部左侧，同一行横向排布，用于标识当前演员 |
| 结果总数 | 辅助信息 | 是 | 无 | `GET /api/actors/{actorId}/videos` | 固定放在主结果区顶部右侧 |
| 关联影片列表 | 卡片结果区 | 是 | 打开影片详情 | `GET /api/actors/{actorId}/videos` | 采用共享 `统一影片卡片` 的宽图卡片网格展示，与影片详情返回链路打通 |
| 分页控件 | 翻页控件 | 是 | 切换当前页 / 跳转目标页 | 当前查询状态 | 位于右下角，复用共享分页控件的 `<`、`当前页 / 总页数`、`>`、`跳转` 结构 |

## 交互规则

- 从 `actors-page` 点击演员卡片进入本页。
- 顶部 `返回演员列表` 需恢复原来的关键字、排序和分页状态。
- 右侧关联影片卡片右键或 `更多` 按钮打开统一的 `video-context-menu`。
- 右侧关联影片卡片点击后进入影片详情页。
- 从影片详情返回时，应先回到当前演员详情页，而不是直接回到 Actors 列表。

## 状态定义

### Loading

- 首次进入本页时显示演员详情和关联影片 loading。

### Empty

- 演员详情缺失时显示空态。
- 关联影片为空时，右侧结果区显示空态。

### Error

- 演员详情或关联影片拉取失败时显示错误提示与刷新入口。

## 性能与体验约束

- 页面主体应优先服务关联影片展示，不保留独立左侧信息栏。
- 演员名称与 `Actor ID` 固定放在主结果区顶部左侧，并以同一行横向排布，不再额外占用单独栏位。
- 页面不再显示 `关联影片` 这一段落标题，避免重复表达页面语义。
- 关联影片结果区按共享 `统一影片卡片` 的视觉结构展示，不再退回为纯文本列表行。
- 影片卡片保持大图主视觉、图底细条与居中文本的轻信息结构，不回退为左图右文块。
- 右下角翻页区复用共享 `分页控件` 样式与交互顺序，避免页面私有翻页样式。
- 页面图只表达结构，不写解释性长文案。

## 回归点

- 能从 Actors 列表进入演员详情页。
- 演员名称与 `Actor ID` 能在主结果区顶部左侧以横向排布正确显示。
- 页面不再出现独立左侧信息栏。
- 页面不再出现 `关联影片` 分区标题。
- 关联影片列表能以统一影片卡片样式正确显示并分页。
- 关联影片动作菜单可正常打开并执行。
- `Actors -> Actor Detail -> Video Detail -> Actor Detail` 返回链路正常。
