# Actor Detail Page Spec

## 页面目的

- `Actor Detail Page` 是演员列表进入后的独立详情页。
- 它负责展示演员头部信息、关联媒体库、来源信息和关联影片结果集。

## 页面范围

- 本页负责：
  - 展示演员头部信息
  - 展示演员关联媒体库
  - 展示来源链接
  - 展示演员关联影片结果集
  - 从关联影片进入影片详情页
  - 提供返回 Actors 列表的链路
- 本页不负责：
  - 演员编辑
  - 媒体库管理
  - 影片详情主内容本身

## 数据来源

- 演员详情：
  - `GET /api/actors/{actorId}`
- 演员关联影片：
  - `GET /api/actors/{actorId}/videos`
- 影片详情：
  - `GET /api/videos/{videoId}`

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 当前线框只表达右侧内容区。
- 内容区采用左右分栏：
  - 左侧为演员头部与属性信息
  - 右侧为关联影片结果区
- 顶部右侧保留返回 Actors 列表按钮。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 页面标题 | 静态标题 | 是 | 无 | 固定文案 | 文案为 `演员详情 / Actor Detail` |
| 返回按钮 | 次按钮 | 是 | 返回 Actors 列表 | 当前返回态 | 恢复原列表状态 |
| 头像区 | 头部信息 | 是 | 无 | `GET /api/actors/{actorId}` | 真实头像或占位头像 |
| 演员名称 / ID | 头部信息 | 是 | 无 | `GET /api/actors/{actorId}` | 展示主标识 |
| 指标区 | 信息块 | 是 | 无 | `GET /api/actors/{actorId}` | 展示作品数、媒体库数、最近扫描、最近播放 |
| 所属媒体库 | 标签区 | 是 | 无 | `GET /api/actors/{actorId}` | 展示库名集合 |
| 来源链接 | 信息块 | 条件 | 打开外部来源 | `GET /api/actors/{actorId}` | 无值时隐藏 |
| 关联影片列表 | 内容区 | 是 | 打开影片详情 | `GET /api/actors/{actorId}/videos` | 与影片详情返回链路打通 |
| 分页控件 | 翻页控件 | 是 | 切换当前页 | 当前查询状态 | 位于右下角 |

## 交互规则

- 从 `actors-page` 点击演员卡片进入本页。
- 顶部 `返回演员列表` 需恢复原来的关键字、排序和分页状态。
- 右侧关联影片项点击后进入影片详情页。
- 从影片详情返回时，应先回到当前演员详情页，而不是直接回到 Actors 列表。

## 状态定义

### Loading

- 首次进入本页时显示演员详情和关联影片 loading。

### Empty

- 演员详情缺失时显示空态。
- 关联影片为空时，右侧结果区显示空态。

### Error

- 演员详情或关联影片拉取失败时显示错误提示与刷新入口。

## 性能与体验约束

- 头部信息与关联影片区明确分栏，不混成单列长表。
- 头像、名称和关键指标必须在首屏可见。
- 页面图只表达结构，不写解释性长文案。

## 回归点

- 能从 Actors 列表进入演员详情页。
- 头部信息能正确显示。
- 关联媒体库信息能正确显示。
- 关联影片列表能正确显示并分页。
- `Actors -> Actor Detail -> Video Detail -> Actor Detail` 返回链路正常。
