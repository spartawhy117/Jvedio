# Main Shell Spec

## 页面目的

- `Main Shell` 定义当前 exe 主窗口的共享壳层。
- 它负责品牌区、一级导航、媒体库入口和右侧内容承载区。

## 页面范围

- 本壳层负责：
  - 左侧固定导航栏
  - 品牌区
  - 一级导航入口
  - 影视库区入口
  - 右侧内容承载区
- 本壳层不负责：
  - 具体业务页面内容布局细节
  - 设置页内部表单
  - 库内容页具体结果集
  - 浏览器地址栏式导航体验

## 布局

- 主窗口使用双栏结构：
  - 左侧固定导航栏
  - 右侧自适应内容区
- 左栏宽度固定，不随窗口拉伸。
- 右侧内容区随窗口尺寸变化而伸缩。
- 当前线框采用 Light 方向。

## 左侧导航结构

### 品牌区

- 左侧为应用 icon。
- 右侧为应用名 `Jvedio Next`。
- 品牌区采用“左 icon + 右名称”的一行结构。

### 一级导航

- `设置`
- `库管理`
- `喜欢`
- `演员`

规则：
- 一级导航全部使用相同的按钮高度、间距和 icon-left + label-right 结构。
- `设置` 位于最上方，样式与 `库管理` 等导航按钮完全一致。

### 影视库区

- 区块标题：`影视库`
- 每个已配置媒体库占一行入口。
- 点击某个库后，右侧内容区切换到该库的内容页。
- 当前版本不保留 `智能分类` 区块，也不提供 `类别 / 系列` 的正式导航入口。

## 页面职责关系

- `库管理`
  - 承担媒体库管理中心职责
  - 负责建库、删库、编辑库和进入单库
- `喜欢`
  - 承担收藏聚合结果浏览
  - 第一轮不扩展新的收藏业务规则
- `演员`
  - 承担跨库演员聚合、筛选、排序和进入演员详情
- `影视库`
  - 承担单库内容浏览入口
  - 点击后进入单库内容页
- `设置`
  - 承担当前可见设置页与 MetaTube diagnostics 入口

## 交互规则

- 点击 `设置` 直接进入设置页，不再通过底部弹出菜单或二级展开入口。
- 点击 `库管理` 进入媒体库管理首页。
- 点击 `喜欢`、`演员` 进入对应一级内容页。
- 点击某个媒体库名称进入该库的内容页。
- 内容页线框只需要表达右侧内容区，不重复绘制整个左栏。

## 状态定义

### Default

- 显示品牌区、一级导航和影视库区。

### Active

- 当前路由对应的导航项显示选中态。

### Empty

- 没有任何媒体库时，`影视库` 区块仍保留标题，但不显示库项列表。

## 性能与体验约束

- 左栏保持稳定，不因右侧页面切换产生结构抖动。
- 主壳层优先提供清晰导航，不承载业务说明性长文本。
- 当前版本不为未接线能力预留左侧导航占位区。
- 不营造浏览器地址栏和多级导航栏体验，保持桌面壳层感。

## 回归点

- 品牌区显示 `Jvedio Next`。
- `设置` 位于最上方且与其它一级导航同规格。
- 左侧导航不再出现 `智能分类 / 类别 / 系列` 入口。
- 进入不同一级页面时，右侧内容区可正常切换。
