# Library Page Spec

## 页面目的

- `Library Page` 表示单个媒体库内部的影片内容页。
- 它负责浏览当前库中的影片，不负责管理“库对象”本身。

## 页面范围

- 本页负责：
  - 展示当前库的影片结果集
  - 提供最小筛选、排序、刷新
  - 打开影片详情页
  - 保留从详情页返回当前库的链路
- 本页不负责：
  - 新建媒体库
  - 删除媒体库
  - 编辑扫描目录
  - 展示库级配置表单

## 数据来源

- 当前库影片结果集：
  - `GET /api/libraries/{libraryId}/videos`
- 当前库任务摘要：
  - `GET /api/tasks`
- 当前库任务与资源变更：
  - `GET /api/events`
- 影片详情跳转目标：
  - `GET /api/videos/{videoId}`

## 主要能力

- 浏览当前库影片结果集
- 进行关键字筛选、排序与分页
- 刷新当前库结果
- 下钻到影片详情
- 承接当前库相关任务反馈

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 当前线框只表达右侧内容区。
- 页面标题使用当前库名称。
- 顶部查询栏复用共享 `结果页查询工具栏`：
  - 左侧为搜索框与刷新按钮
  - 右侧为排序下拉按钮
  - 选择排序项后立即执行排序
- 当前库影片结果默认支持：
  - `按VID顺序 / 按VID倒序`
  - `按名称顺序 / 按名称倒序`
  - `按发布日期顺序 / 按发布日期倒序`
  - `按导入时间顺序 / 按导入时间倒序`
- 主体区域使用共享影片卡片网格。
- 单张卡片对齐共享 `统一影片卡片`：
  - 宽图主视觉优先展示影片海报或主图
  - 右上角保留勾选按钮
  - 图底保留细状态条
  - 下方文字区只保留 `VID + 标题短名` 与 `年月`
- 分页控件位于内容区右下角。

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 页面标题 | 静态标题 | 是 | 无 | 当前库 | 显示当前库名 |
| 查询工具栏 | 组合工具栏 | 是 | 搜索、刷新、切换排序 | 当前查询状态 | 复用共享 `结果页查询工具栏` |
| 影片卡片网格 | 内容区 | 是 | 点击进入影片详情 | `GET /api/libraries/{libraryId}/videos` | 复用统一影片卡片语义 |
| 单张影片卡片 | 卡片 | 是 | 打开影片详情 | 当前列表项 | 显示右上角勾选、`VID + 标题短名`、年月与主图底部细状态条 |
| 分页控件 | 翻页控件 | 是 | 切换当前页 | 当前查询状态 | 放在右下角 |

## 交互规则

- 进入方式：
  - 从左侧库导航进入
  - 从库管理页的 `打开` 进入
- 当前页只作用于一个库，不跨库聚合。
- 搜索框提交关键字后按当前库范围刷新结果集。
- 点击刷新按钮时按当前关键字、排序和分页上下文重新请求结果集。
- 打开排序下拉后，点击任一排序项立即执行排序并关闭下拉层。
- 影片卡片右键或 `更多` 按钮打开统一的 `video-context-menu`。
- 勾选多张影片卡后，右键当前选中卡片或选中区域，打开统一的 `video-batch-context-menu`。
- 影片卡片点击后进入影片详情页。
- 影片详情返回时应恢复当前库页的筛选、排序和分页状态。
- 当前线框阶段不额外增加顶部右侧动作区。

## 当前实现边界

- 当前页优先聚焦“单库结果浏览 + 详情返回链路 + 任务反馈”。
- 扫描与抓取闭环仍属于当前单库工作台语义，但不要求在文档迁移时新增新的按钮区或配置区。
- 不扩展多库联合视图、高级批量编辑或复杂统计面板。

## 状态定义

### Loading

- 首次进入当前库时显示结果集 loading。

### Empty

- 当前库无影片时显示轻量空态。
- 空态文案应引导用户先回到库管理页执行扫描。

### Error

- 拉取结果集失败时显示错误提示，重新请求统一通过查询栏 `刷新` 触发。

## 性能与体验约束

- 首屏优先展示当前页结果，不预加载全部影片。
- 默认信息密度保持接近 Favorites 页。
- 影片卡片必须与共享 `统一影片卡片` 保持一致，不再回退到竖向封面卡或多行信息卡。
- 当前阶段优先保持浏览稳定，不在此页混入库编辑控件。
- 当前页路由状态应继续承载筛选、排序和分页，保证从详情页返回后可恢复浏览上下文。

## 回归点

- 能正确进入指定库内容页。
- 能正确显示当前库影片结果集。
- 查询栏搜索、刷新与排序切换有效。
- 分页切换有效。
- 影片卡片右键动作菜单可正常打开并执行。
- 多选影片后的批量动作菜单可正常打开并执行。
- 打开影片详情后返回链路正常。

## 命名约束

- `library-page` 是当前正式的“库内容页”名称。
- 旧的 `library-page-content` 不再作为独立页面名称保留。
