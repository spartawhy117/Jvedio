# Library Management Page Spec

## 页面目的

- `库管理` 是当前 exe 的媒体库管理首页。
- 它负责管理“库对象”本身，不负责展示某个库里的影片内容。
- 这一页替代旧启动窗口，承担日常建库、删库、扫描和编辑入口。

## 页面范围

- 本页负责：
  - 新建媒体库
  - 查看库列表
  - 打开库内容页
  - 触发单库扫描
  - 打开编辑弹窗
  - 删除媒体库
- 本页不负责：
  - 展示影片卡片
  - 编辑高级扫描策略
  - 处理库配置导入 / 恢复
  - 展示扫描目录明细列表

## 数据来源

- 媒体库列表：
  - `GET /api/libraries`
- 新建媒体库：
  - `POST /api/libraries`
- 编辑媒体库：
  - `PUT /api/libraries/{libraryId}`
- 删除媒体库：
  - `DELETE /api/libraries/{libraryId}`
- 触发扫描：
  - `POST /api/libraries/{libraryId}/scan`
- 任务状态：
  - 全局任务摘要和事件流
  - `GET /api/tasks`
  - `GET /api/events`

## 主要能力

- 新建媒体库
- 删除媒体库
- 编辑媒体库名称与扫描目录
- 打开单库内容页
- 查看库基础状态与最近任务摘要
- 触发单库扫描

## 布局

- 本页复用 `main-shell.md` 定义的共享壳层。
- 当前线框只表达右侧内容区。
- 页面顶部只保留一个主按钮：
  - `新建媒体库`
- 主体区域使用列表式布局：
  - 一行一个媒体库
  - 不使用海报卡片式库展示

## 列表字段

- `媒体库`
  - 显示库名
  - 可附一行简短状态说明
- `影片数`
  - 显示当前库内影片总数
- `最近扫描`
  - 显示最近一次扫描完成时间
- `状态`
  - 使用 badge 显示简短状态
- `操作`
  - 固定为单行 action strip

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 页面标题 | 静态标题 | 是 | 无 | 固定文案 | 文案为 `库管理 / Library Management` |
| `新建媒体库` | 主按钮 | 是 | 打开新建弹窗 | 无 | 顶部唯一主操作 |
| 库名称 | 行内主文本 | 是 | 可点击进入库内容页 | `GET /api/libraries` | 行主入口 |
| 简短状态说明 | 次级文本 | 是 | 无 | 任务状态 + 本地状态映射 | 用一行短文案，不展示扫描目录 |
| 影片数 | 文本 | 是 | 无 | `GET /api/libraries` | 只显示总数 |
| 最近扫描 | 文本 | 是 | 无 | `GET /api/libraries` | 无值时显示未记录 |
| 状态 badge | badge | 是 | 无 | 扫描状态映射 | 只显示固定状态词 |

| `删除` | 危险按钮 | 是 | 打开删除确认 | 当前库数据 | 二次确认后删除 |

## 交互规则

- 行主入口：
  - 点击库名或 `打开` 进入该库内容页
- 行动作收口：
  - 所有库级操作都放在同一行右侧 action strip
  - 不再拆出顶部次按钮区
- 编辑入口：
  - 点击 `编辑` 打开弹窗
  - 弹窗复用现有新建媒体库弹窗壳
  - 打开时预填 `媒体库名称` 和 `扫描目录`
  - 提交走更新接口
- 扫描入口：
  - 点击 `扫描` 启动当前库扫描任务
  - 扫描中按钮文案切换为进度值，例如 `42%`
  - 同步更新行状态 badge
- 删除入口：
  - 点击 `删除` 先打开确认弹窗
  - 删除确认弹窗复用 `confirm-dialog` 骨架，正文区注入当前库摘要与影响说明
  - 确认后才调用删除接口
- 新建入口：
  - 点击 `新建媒体库` 打开新建弹窗
  - 弹窗字段与编辑弹窗保持同一结构

## 当前实现边界

- 当前页优先承担媒体库 CRUD 与进入单库内容页的闭环。
- 当前页展示任务摘要，但不展开完整任务中心。
- 当前页不承接复杂封面管理、统计图表或导入恢复流程。
- 扫描链路以“列表行状态 + 全局活动条 + 详情弹窗”协同反馈，不要求在本页内增加额外配置面板。

## 状态定义

### Loading

- 首屏拉取库列表时显示列表骨架或占位区。
- 顶部 `新建媒体库` 保持可点。

### Empty

- 当没有任何媒体库时：
  - 显示空态说明
  - 只保留一个主按钮 `新建媒体库`
- 空态不显示 `导入 / 恢复`

### Error

- 库列表拉取失败时显示错误提示和重试入口。
- 单行动作失败时只反馈当前行，不阻塞整个页面。

### Running

- 扫描进行中：
  - `扫描` 按钮显示进度
  - `状态` badge 同步显示进行中状态

### Disabled

- 扫描中的同一行可按实现需要禁用重复扫描。
- 删除中的同一行临时禁用重复提交。

## 状态 badge 规范

- 固定只使用以下 3 类：
  - `待扫描`
  - `扫描中 xx%`
  - `已同步`
- 颜色建议：
  - `待扫描`：绿色
  - `扫描中 xx%`：蓝色
  - `已同步`：灰色

## 编辑弹窗规则

- 第一版直接复用 `新建媒体库` 弹窗结构。
- 字段：
  - `媒体库名称`
  - `扫描目录`
- 列表页不展示扫描目录。
- 扫描目录只在编辑弹窗内出现，避免把列表页做成配置表。

## 当前实现约束

- 当前 `库管理` 页不保留 `导入 / 恢复` 入口。
- 原因：
  - Electron / Worker 当前只有影片扫描导入链
  - 没有完整可用的库配置导入 / 数据恢复产品链路
  - 旧 WPF 的 `RestoreDatabase` 是删除 `app_datas.sqlite` 后重建，不是用户可用的恢复产品
  - 旧 WPF 的 `ExportToNFO` 入口当前没有真实导出实现
  - 首页摘要与任务联动统一通过 Worker `bootstrap / libraries / tasks / events` 获取

## 性能与体验约束

- 首屏只拉库列表与必要任务摘要，不预拉库内容页数据。
- 列表页不承载扫描目录全文，避免高噪音信息占据首屏。
- 单行动作执行后优先局部刷新当前行，再按需要回刷整页。
- 进入页面时可以快速判断是否存在待扫描状态，但不要在此页做重量级目录遍历展示。

## 回归点

- 库列表可正常展示。
- `新建媒体库` 可打开并成功创建。
- `编辑` 可复用弹窗并成功保存。
- `删除` 有确认且能正确删除。
- `扫描` 可触发任务并显示进行中状态。
- 进入库内容页和返回链路正常。
- 空态页不出现 `导入 / 恢复`。
