# Delete Library Dialog Spec

## 页面目的

- `Delete Library Dialog` 是媒体库删除确认弹层。
- 它负责在删除前明确影响范围，并进行二次确认。

## 页面范围

- 本弹层负责：
  - 展示当前待删除库的摘要信息
  - 明确删除影响范围
  - 展示删除失败反馈
  - 确认执行删除
- 本弹层不负责：
  - 删除磁盘上的影片文件
  - 展示库内影片详情
  - 处理批量删除

## 数据来源

- 删除媒体库：
  - `DELETE /api/libraries/{libraryId}`

## 布局

- 使用标准危险操作模态弹层结构。
- 顶部保留标题和关闭按钮。
- 主体分为：
  - 影响说明
  - 当前库摘要
  - 错误反馈区
- 底部按钮顺序固定：
  - 左侧 `取消`
  - 右侧危险按钮 `确认删除`

## 元素清单

| 元素 | 类型 | 常显 | 行为 | 数据来源 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 标题 | 静态标题 | 是 | 无 | 固定文案 | 文案为 `删除媒体库` |
| 关闭按钮 | 图标按钮 | 是 | 关闭弹层 | 无 | 右上角关闭 |
| 影响说明 | 警示文本 | 是 | 无 | 固定规则 | 明确不删除磁盘影片文件 |
| 库摘要 | 信息块 | 是 | 无 | 当前库数据 | 展示名称、主路径、影片数 |
| 错误反馈区 | 提示条 | 条件 | 展示删除失败 | 接口返回 | 无错误时隐藏 |
| 取消按钮 | 次按钮 | 是 | 关闭弹层 | 无 | 不提交 |
| 确认删除 | 危险按钮 | 是 | 调用删除接口 | `DELETE /api/libraries/{libraryId}` | pending 时禁用 |

## 交互规则

- 从 `library-management-page` 的 `删除` 动作进入。
- 删除前必须展示当前库摘要。
- 确认后才真正调用删除接口。
- 删除失败时在弹层内联反馈，不跳转页面。

## 状态定义

### Confirm

- 默认状态，等待用户确认或取消。

### Pending

- 删除中时禁用重复提交。

### Error

- 删除失败时显示错误反馈区。

## 性能与体验约束

- 删除弹层必须保持短路径、强确认，不承载额外信息。
- 危险操作按钮始终使用明确红色语义。

## 回归点

- 点击 `删除` 能打开确认弹层。
- 摘要信息显示正确。
- 确认删除后能成功删除库。
- 删除失败时有明确反馈。
