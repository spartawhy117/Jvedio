## 当前已确认的参考点

- 桌面壳思路：
  - Electron 作为桌面前端容器
- 页面组织：
  - Web 前端承载主要页面内容
- 桌面交互感：
  - 强调本地客户端体验而不是浏览器站点感
- 页面密度：
  - 偏桌面工具型信息密度，而非大留白展示型布局
- 桌面增强方式：
  - 托盘、无边框窗口、自定义标题栏、本地播放器调起、独立代理进程生命周期管理

## 审计后新增结论

- 该项目不是“本地自建页面前端”范式。
- 其主体模式更接近：
  - Electron 主壳
  - 本地登录页
  - 远端 Web 页面加载
  - preload DOM 注入
  - IPC 驱动本地播放器/代理/托盘增强
- 因此它适合作为：
  - 桌面壳参考
  - 桌面交互参考
  - 本地能力挂接参考
- 不适合作为：
  - Jvedio 页面组件拆分模板
  - Jvedio Router / 状态管理 / 页面代码组织模板
  - Jvedio Worker API 设计模板

## 与 `jellyfin-web` 的分工

- `fntv-electron`
  - 负责桌面壳层参考
- `jellyfin-web`
  - 负责页面结构与数据分层参考
- Jvedio 后续应避免继续把 `fntv-electron` 当作页面实现模板。
