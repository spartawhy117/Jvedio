## 当前收口结果
说明：
- 版本更新后新增的 `F-038` 至 `F-042` 已完成代码收口，并已回跑本轮自动化验证。
- 真包场景仍建议优先人工复核播放链路与 4K 详情页观感，但当前不再作为阻断发布的活跃问题保留。

### F-038 播放接口回执与诊断日志

- 已统一播放请求 / 响应契约：后端与前端都只保留 `playerPath`、`played`、`playerUsed` 三个现行字段。
- `VideoService` 已补充播放日志，记录文件路径、文件大小、请求 / 配置 / 实际解析出的播放器路径，以及系统默认回退开关，便于真包手动回看 `log/runtime/`。
- 自动化后端校验已改为在 E2E 假样本环境中显式使用 `cmd.exe` 作为测试播放器，避免空视频样本把“接口契约校验”误判成播放器能力失败。
- 真包复核后又确认一处真实链路差异：Windows 通过系统默认播放器打开 `.mp4` 时，`Process.Start(...)` 可能返回 `null`，但播放器本身已经成功接管；当前实现已改为把这种 shell handoff 视为成功，不再误报 HTTP 500。
- 当前结论：`/api/videos/{id}/play` 已恢复为 `200 + played=true` 的稳定返回；真包真实影片播放仍建议人工抽检。

### F-039 详情页 4K 布局与 fanart 层级

- 已将详情页重构为宽屏 hero 结构，取消固定左侧海报列，顶部主视觉改为 fanart 背景层。
- `NFO / 海报 / 缩略图 / fanart` 状态 badge 已移到标题区下方横向排布，不再挂在 `VID` 下方。
- 主动作区、元数据区、路径区和演员区已重新分层，大屏下不再被旧的中屏双栏结构限制。
- 当前结论：前端构建通过，详情页 DOM 与样式结构已按新口径收口。

### F-040 设置页底部动作区锚定

- 已将设置页底部动作按钮从限宽表单流中拆出，接入全宽 footer 容器。
- 正文内容仍保留阅读限宽，但底部动作区已基于整个活动页宽度右对齐。
- 当前结论：设置页布局已满足高分辨率右下角锚定要求，前端构建通过。

### F-041 运行状态 CPU / 内存口径

- Tauri 侧运行状态采样已改为聚合 shell 进程树、Worker 进程树和相关子进程，并避免对 Worker 重复计数。
- CPU 总量已按逻辑 CPU 数做归一化，Memory 总量改为展示前端整体与 Worker 的合计占用。
- 当前结论：Rust `cargo check` 已通过，运行状态展示口径已与产品要求对齐。

### F-042 演员头像缓存命名与复用

- 演员头像缓存已改为优先使用“演员展示名规范化”作为主文件名，提升 `cache/actor-avatar/` 可读性。
- 读取链路保持对旧 `actorId / url / hash` 命名的兼容，并会在命中旧缓存时尝试迁移到新的演员名路径。
- 缓存生命周期保持“长期保留不删除”，重新索引相同演员时优先复用本地缓存，避免重复下载。
- 当前结论：已补充头像缓存命名与迁移测试，验证新旧缓存路径都能命中或被提升。
