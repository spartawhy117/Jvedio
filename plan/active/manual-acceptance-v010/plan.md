# v0.1.0 人工验收计划

## 目标

在 `v0.1.0` 便携版（`JvedioNext_0.1.0_x64-portable.zip`）上完成一轮**端到端人工验收**，覆盖从解压启动到日常使用的全部核心流程，确认真实用户体验无阻断性问题后，再启动下一版功能规划。

## 背景

- Phase 10 自动化 E2E 已跑通 7 组 flow（Playwright MCP + 浏览器模式）
- 62 个后端测试全部通过
- 但自动化验收运行在浏览器模式，**尚未在正式发布包环境下做过人工全链路验证**
- 本次验收的目标环境是：**解压 ZIP 便携版 → 双击 `JvedioNext.exe` → 真实 Tauri 壳层**

## 执行方式说明


### 标记说明

- `[自动化]`：优先通过现有脚本、后端测试或 Playwright MCP 浏览器模式完成
- `[人工]`：必须在真实发布包 / 真实桌面环境下操作确认
- `[混合]`：自动化先给出基线，最终仍需在真实 exe 场景复核

### 推荐执行顺序（先自动化，再人工）

**不要按条目逐个切环境。** 推荐始终按两轮执行：

1. **第一轮：自动化基线**
   - 统一完成全部 `[自动化]` 项
   - 同时完成全部 `[混合]` 项里的“自动化部分”
2. **第二轮：真实桌面复核**
   - 统一完成全部 `[人工]` 项
   - 同时完成全部 `[混合]` 项里的“桌面复核部分”

这样可以避免在浏览器模式、脚本环境、真实 exe 环境之间来回切换。

### 第一轮：自动化基线（浏览器模式）

按以下顺序执行：

1. `scripts/seed-e2e-data.ps1 -SkipWorkerShutdown -NoPause`
2. `scripts/verify-backend-apis.ps1 -NoPause`
3. `scripts/start-e2e-env.ps1 -NoPause`
4. 打开 `test-data/e2e/e2e-env.json`，读取 `baseUrl`
5. 用 Playwright MCP 打开 `http://localhost:1420?workerPort={port}` 或 `http://localhost:1420?workerUrl=http://127.0.0.1:{port}`
6. 按以下顺序执行验收：
   - `Phase 2` 主壳导航
   - `Phase 3` 库管理
   - `Phase 4` 扫描与目录整理
   - `Phase 5` 搜刮（MetaTube）
   - `Phase 6` 单库浏览
   - `Phase 8` 收藏与演员
   - `Phase 1 / 7 / 9 / 10` 中标记为 `[自动化]` 的条目
7. 对 `[混合]` 项先记录“自动化基线”结果
8. 验收后运行 `scripts/stop-e2e-env.ps1 -NoPause`；若本轮已经改动测试数据，继续执行 `scripts/cleanup-e2e-data.ps1 -NoPause` 重置到基线

自动化阶段的主要目标：

- 验证页面流转、列表刷新、搜索 / 排序 / 分页、弹层、右键菜单、多选批量动作
- 验证设置保存、SSE 回流、失败样本显示、单影片 / 批量重抓反馈
- 验证后端链路基线未回归（扫描、导入、搜刮、sidecar 写出、设置持久化）

### 第二轮：真实桌面复核（发布包 / exe）

按以下顺序执行：

1. 执行 `pwsh -ExecutionPolicy Bypass -File scripts/build-release.ps1`
2. 解压 `build/release/JvedioNext_0.1.0_x64-portable.zip`
3. 双击 `JvedioNext.exe`
4. 按以下顺序执行验收：
   - `Phase 1` 中标记为 `[人工]` / `[混合]` 的条目
   - `Phase 7` 中标记为 `[人工]` 的条目
   - `Phase 9` 中标记为 `[人工]` 的条目
   - `Phase 10` 中标记为 `[人工]` 的条目
5. 对 `[混合]` 项补记录“桌面复核”结果，并给出最终结论
6. 将问题记录到本文档的“发现问题记录格式”表格

人工阶段的主要目标：

- 验证真实 Tauri 壳层启动、Worker 自动拉起、单实例控制
- 验证系统级能力：默认播放器、资源管理器、系统浏览器
- 验证重启后设置是否保持

### 混合项记录方式

对每个 `[混合]` 项，不要只记录一个“通过 / 失败”，而是拆成三段：

1. **自动化基线**：浏览器模式下是否通过
2. **桌面复核**：真实 `JvedioNext.exe` 环境下是否通过
3. **最终结论**：仅当两段都通过时，才记为最终通过

示例：

- `1.3 Worker 自动拉起`
  - 自动化基线：通过（`WorkerStatusOverlay` 出现后消失）
  - 桌面复核：通过（双击 `JvedioNext.exe` 后无需手动启动 Worker 即进入主界面）
  - 最终结论：通过

3. `[混合]` 观察 `WorkerStatusOverlay` 是否先出现再消失
4. `[人工]` 再次双击 `JvedioNext.exe`，确认不会启动第二个窗口
