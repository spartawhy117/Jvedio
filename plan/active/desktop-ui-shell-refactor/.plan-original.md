# Desktop UI Shell Refactor

## Initial Request Snapshot

- Persist the UI refactor planning work before implementation.
- Use `plan/active/desktop-ui-shell-refactor/` as the only active feature workspace.
- Use `doc/UI/desktop-ui-shell-refactor/` as the stable design asset directory.
- Do not implement UI code yet; prepare planning docs first, then produce drawings after a drawing MCP is installed.

## Confirmed Product Decisions

- Main window remains the only regular entry point.
- The old startup window is not a daily library selector anymore.
- Home page becomes a media-library management page instead of a visual library poster wall.
- Left navigation is fixed to:
  - Home
  - Favorites
  - Actors
  - Media libraries
  - Categories
- Categories only expose existing `Genre` and `Series`.
- Settings stay in a separate window.
- Settings language choices are limited to `中文` and `English`.
- Settings theme choices are limited to `Light` and `Dark`.
- First run no longer shows a dedicated language/theme picker dialog.

## External Reference Constraints

- `fntv-electron` is a visual and information-architecture reference only.
- Do not directly copy its code, stylesheets, bundled assets, or brand resources.
- Recreate the intended structure and visual language in WPF.
