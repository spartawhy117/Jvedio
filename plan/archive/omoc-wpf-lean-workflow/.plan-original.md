# Baseline Plan Draft

## Feature
- slug: `omoc-wpf-lean-workflow`
- title: `Oh My OpenCode lean workflow for WPF testing and UI refactor`

## Requirement Summary
- Produce a concise but complete Oh My OpenCode usage plan for a WPF project.
- Minimize agent count, dispatch frequency, and iteration loops because only one LLM subscription is available.
- Cover both test work and UI refactor work.
- Deliver the result as a markdown guide.

## Repository Context
- Project: `D:\study\Proj\Jvedio`
- WPF app: `Jvedio-WPF/Jvedio`
- Test project: `Jvedio-WPF/Jvedio.Test`
- Main UI entry: `Jvedio-WPF/Jvedio/Windows/Window_Main.xaml` + `Window_Main.xaml.cs`
- Main VM entry: `Jvedio-WPF/Jvedio/ViewModels/VieModel_Main.cs`
- Style entry: `Jvedio-WPF/Jvedio/CustomStyle/StyleManager.cs`

## Initial Recommendation
- Prefer a single primary agent workflow.
- Keep planning and execution in the same conversation context.
- Disable routine subagent fan-out.
- Limit every phase to fixed review caps.

## Early Risks
- Mixing UI refactor and test expansion in one batch may increase iteration cost.
- Large XAML files like `Window_Main.xaml` and `VideoList.xaml` can consume context quickly.
- Integration tests with real MetaTube traffic are costlier and slower than unit tests.
