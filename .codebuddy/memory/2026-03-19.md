## Phase 4: Release 切换与旧启动链替换 — 全部完成 ✅

### 已完成子步骤

- **4.1** `App.xaml.cs` — 将 `ElectronShellLauncher` 类替换为 `TauriShellLauncher`
  - Release 模式下 `OnStartup` 优先尝试启动 `tauri-shell/Jvedio.exe`
  - 失败时 fallback 到 WPF 旧启动链
  - `JVEDIO_FORCE_LEGACY_WPF` 环境变量继续可用
- **4.2** `Jvedio.csproj` — 将 `PrepareElectronShellArtifacts` Target 替换为 `PrepareTauriShellArtifacts`
  - 构建 Worker → 调用 `npm run tauri build` → 复制产物到 `tauri-shell/` 和 `worker/`
  - 不再依赖 `electron/node_modules/electron/dist/electron.exe`
- **4.3** Tauri bundle 配置
  - `tauri.conf.json` 添加 `bundle.resources` 将 `worker-dist/**/*` 映射到安装包 `worker/`
  - 新建 `tauri/scripts/prepare-worker.ps1` — Worker 预构建脚本
  - `tauri/package.json` 添加 `prepare-worker` + `build:release` 脚本
  - `tauri/.gitignore` 添加 `worker-dist` 忽略规则
- **4.4** `electron/README.md` 更新为 DEPRECATED 标记
- **4.5** 更新 `handoff.md` + `validation.md` 文档

### Release 构建链路（新）
```
Jvedio.csproj (Release) → PrepareTauriShellArtifacts
  ├─ dotnet build Jvedio.Worker
  ├─ npm run tauri build (在 tauri/ 下)
  └─ 复制产物到 bin/Release/tauri-shell/ 和 bin/Release/worker/

独立 Tauri 构建：
  cd tauri && npm run build:release
  ├─ prepare-worker (PowerShell → 构建 Worker → 复制到 worker-dist/)
  └─ tauri build (前端构建 + Rust 编译 + bundle → msi/nsis)
```
