## Phase 1.2: 壳层拉起 Jvedio.Worker 并注入动态 baseUrl ✅

### Rust 端 (src-tauri)
- 新增 `worker.rs` 模块：Worker 子进程生命周期管理。
- `spawn_worker()` 在 `setup()` 中启动 Worker 子进程，监听 stdout 解析 `JVEDIO_WORKER_READY <baseUrl>` 就绪信号。
- 解析到 baseUrl 后通过 `app.emit("worker-ready", { baseUrl })` 发送到 renderer。
- Worker 异常退出或启动失败时发送 `worker-error` 事件。
- Worker 日志（stdout/stderr）转发为 `worker-log` 事件。
- `kill_worker()` 在主窗口关闭时终止子进程。
- Tauri command `get_worker_base_url` 供 React 端同步查询已就绪的 baseUrl。
- `WorkerState` 管理子进程句柄和 baseUrl（`Mutex` 保护）。
- dev 模式从 workspace 相对路径定位 Worker exe；bundled 模式从 exe 同目录 `worker/` 下定位。

### React 端 (src/)
- 新增 `contexts/WorkerContext.tsx`：`WorkerProvider` + `useWorker()` hook。
  - 监听 `worker-ready` / `worker-error` Tauri 事件。
  - 启动时主动 `invoke("get_worker_base_url")` 检查快启动情况。
  - 暴露 `{ status, baseUrl, error }` 三元组。
- 新增 `components/WorkerStatusOverlay.tsx`：全屏遮罩。
  - status=starting → spinner + "正在启动引擎…"
  - status=error → 错误图标 + 错误详情
  - status=ready → 自动隐藏
- App.tsx 集成：
  - 侧边栏底部 Worker 状态指示器（绿色/橙色/红色圆点 + 文字）。
  - 内容区显示当前 Worker baseUrl（调试用）。

## Phase 1.5: light/dark 主题 + zh/en 多语言 + 资源接线骨架 ✅
